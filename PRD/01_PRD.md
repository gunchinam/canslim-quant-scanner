## 1. 제품 개요

### 한 줄 요약
종목 스캐너 웹앱에서 사용자가 실제로 "한줄평"이라고 인식하는 화면(판단 포스터의 큰 헤드라인)에 진짜 한줄평 엔진(`one_liner.py`)의 결과를 연결하고, 모든 한줄평 문구를 "명사형 종결" 문체로 통일한다.

### 해결하는 문제
직전 골("한줄평 엔진 톤 통일 리라이트")에서 `one_liner.py`의 문구 3533개를 전부 주갤밈 톤으로 리라이트했으나, 사용자가 실제로 매일 보며 "한줄평"이라 부르는 화면은 그게 아니었다. 스캐너 목록에서 종목을 클릭하면 뜨는 드로워의 **판단 포스터**(`dp-verdict-poster`, 큰 주황/빨강 헤드라인)가 진짜 눈에 띄는 자리였는데, 이건 `one_liner.py`와 무관한 별도 로직(`app.js`의 `_renderEntryVerdict`, conv 점수 기반 15단계 × 5개 고정 문구 풀)이 채우고 있었다. 그래서 아무리 `one_liner.py`를 다듬어도 사용자 눈에는 "긍정 눈팅" 같은 뻔한 문구만 반복되는 것으로 보였다.

### 핵심 가치
- 지난 골에서 힘들게 완성한 3533개 리치한 한줄평 문구 풀을, 실제로 가장 눈에 띄는 자리(판단 포스터 헤드라인)에 연결해 드디어 제 역할을 하게 만든다.
- 추가로 문체를 "명사형 종결"(예: "이거 매수하면 주갤에서 박제당함")로 통일해, 짧고 강렬하게 끊어지는 느낌을 준다.

## 2. 사용자

### 주요 사용자
- **누구**: 종목 스캐너 앱을 실사용하는 개인 투자자(제작자 본인 포함, 소수 사용자) — 이전 PRD들과 동일.
- **상황**: 스캐너 목록에서 종목 카드를 클릭해 드로워를 열고, 큰 헤드라인(판단 포스터)을 가장 먼저 본다.
- **목표**: 그 헤드라인 한눈에 "이 종목이 어떤 종목이고 어떤 평가를 받는지" 감이 오길 원한다.

### 사용자 시나리오
1. 사용자가 스캐너 목록에서 종목 카드를 클릭하면
2. 드로워의 판단 포스터가 열리고, 진입 타이밍에 따른 배경색(초록/노랑/빨강)과 확신도(%)는 그대로 뜨지만
3. 그 안의 큰 헤드라인 텍스트는 이제 종목 성격을 압축한 한줄평("이거 매수하면 주갤에서 박제당함" 같은 명사형 종결 문장)으로 바뀌어 있다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
종목 카드 클릭 -> 드로워 오픈 -> 판단 포스터 렌더(conv 기반 색상 + d.OneLiner 헤드라인) -> 한눈에 "어떤 종목·어떤 평가" 인지
```

### 상세 흐름
1. **드로워 오픈**: 스캐너 목록에서 종목 클릭.
2. **판단 포스터 렌더**: `_renderEntryVerdict(d)`가 `conv`로 배경색·확신도%는 그대로 계산하되, 헤드라인 텍스트는 `d.OneLiner`(명사형 종결로 재정비된 한줄평)를 그대로 표시.
3. **길이 대응**: 문구가 길어도 CSS `clamp()`로 폰트가 자동으로 줄어들어 카드 안에 다 표시됨.
4. **일관된 인상**: 어떤 종목을 봐도 "이 종목의 성격"을 압축한 강한 한 줄이 크게 보임.

## 5. 성공 기준

- [ ] `dp-verdict-poster`(모바일)와 `dp-hero-banner-dt`(데스크톱) 헤드라인 자리에 `d.OneLiner` 값이 표시된다(기존 `_pvWord` 5개 고정 문구 풀은 더 이상 헤드라인에 쓰이지 않음).
- [ ] `_pvReason`(3줄 이유), `conv` 계산 로직, 배경색(`_pgCls`)/확신도(%) 표시는 변경 전과 동일하게 동작한다.
- [ ] 헤드라인에 긴 한줄평이 들어가도 카드 레이아웃이 깨지거나 텍스트가 잘리지 않는다(데스크톱·모바일 두 뷰포트 확인).
- [ ] 18개 버킷 3533개 문구 전부가 명사형 종결로 재정비되고, 개수·버킷·극성은 리라이트 전과 정확히 동일하게 유지된다.
- [ ] `tests/test_one_liner_consistency.py`가 이전 골 종료 시점(2 failed/24 passed, 기존 버킷분류 버그만 남은 상태)과 동일하거나 더 나은 결과를 유지한다.
- [ ] `web_app/tests/` 전체 회귀가 기존 베이스라인(195 passed/10 failed)과 동일하게 유지된다.
