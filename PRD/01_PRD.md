# 판단 포스터 한줄평 헤드라인 레이아웃 재설계 -- PRD (Product Requirements Document)

> 생성일: 2026-07-10
> 생성 도구: Show Me The PRD (기존 종목 스캐너 웹앱 내 유지보수 작업)

## 1. 제품 개요

### 한 줄 요약
종목 스캐너 드로워의 판단 포스터 헤드라인이 한줄평(15~40자 문장)을 담기엔 너무 작게 표시되는 문제를, 폰트 축소가 아니라 레이아웃 구조 자체를 재설계해 해결한다.

### 해결하는 문제
직전 골("판단 포스터 한줄평 연결 + 명사형 종결 재정비", 아카이브: `docs/superpowers/godmode-archive/2026-07-10-verdict-poster-oneliner/`)에서 판단 포스터 헤드라인 자리(`_pvWord`, 원래 2~6글자 고정 단어용 초대형 폰트 슬롯)를 한줄평(`d.OneLiner`)으로 교체했다. 슬롯 크기는 그대로 두고 문구 길이에 비례해 폰트만 줄이는 방식(`_dvpWordPx`/`_dhbWordPx`, `app.js:3754-3756`)을 적용했더니, 실사용에서 "한줄평 글자가 너무 작아서 뭐라고 하는지 모르겠다"는 가독성 문제가 발생했다.

### 핵심 가치
"헤드라인 = 한줄평"이라는 직전 골의 방향은 유지하되, 슬롯(카드/컬럼) 자체를 문장 길이에 맞게 넉넉히 재설계해 최소 폰트를 큰 폭으로 끌어올린다. 모바일과 데스크톱은 화면 특성이 달라 각각 별도로 최적화한다.

---

## 2. 사용자

### 주요 사용자
- **누구**: 종목 스캐너 웹앱에서 드로워를 열어 개별 종목을 살펴보는 모든 사용자
- **상황**: 모바일(세로 스크롤 드로워)과 데스크톱(가로형 히어로 배너) 양쪽에서 사용
- **목표**: 판단 포스터 헤드라인(한줄평)을 한눈에, 힘들이지 않고 읽고 싶다

### 사용자 시나리오
1. 사용자가 종목 카드를 눌러 드로워를 열고
2. 판단 포스터의 배경색(초록/노랑/빨강)과 확신도%를 먼저 훑어본 뒤
3. 헤드라인(한줄평)을 읽어 이 종목이 어떤 평가를 받는지 바로 이해한다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
드로워 열기 -> 판단 포스터 확인 -> 헤드라인(한줄평) 읽기 -> 확신도%/이유 확인
```

### 상세 흐름
1. **드로워 열기**: 사용자가 종목 카드를 클릭하면 `dp-verdict-poster`(모바일) 또는 `dp-hero-banner-dt`(데스크톱)가 렌더링된다.
2. **헤드라인 읽기**: `d.OneLiner` 값이 재설계된 레이아웃 안에서 최소 18px 이상 폰트로, 2~3줄 이내로 자연스럽게 표시된다.
3. **부가 정보 확인**: 배경색(`_pgCls`)·확신도%(`conv`)·3줄 이유(`_pvReason`)는 기존 그대로 동작한다.

## 5. 성공 기준

- [ ] 모바일 헤드라인(`dp-verdict-poster` 내 `.dvp-word`)이 40자 한줄평에서도 최소 18px 이상의 폰트를 유지한다.
- [ ] 데스크톱 헤드라인(`dp-hero-banner-dt` 내 `.dhb-word`)이 40자 한줄평에서도 모바일과 같거나 더 큰 최소 폰트를 유지한다.
- [ ] 두 레이아웃 모두 2~3줄 이내로 자연스럽게 줄바꿈되며, 카드 레이아웃 붕괴·텍스트 잘림·가로 스크롤이 발생하지 않는다.
- [ ] 배경색(`_pgCls`)·확신도%(`conv`)·`_pvReason`(3줄 이유)·신호등 라벨 로직과 출력이 변경 전과 동일하다.
- [ ] 짧은 한줄평(15자 내외)과 긴 한줄평(40자 근접) 양쪽 모두 스크린샷으로 레이아웃 검증한다.
- [ ] `tests/test_one_liner_consistency.py` 베이스라인(2 failed/24 passed) 유지.
- [ ] `web_app/tests/`(`test_history_timeline.py` 제외) 195 passed 베이스라인 유지.

## 6. 안 만드는 것 (Out of Scope)

- 한줄평 문구 자체(`one_liner.py`, 3533개) 재작성 -- 이유: 직전 골에서 이미 완료, 이번 골은 순수 레이아웃/CSS/JS 문제.
- `conv`(확신도) 계산 로직, `_pgCls`(배경색 등급), `_pvReason`(3줄 이유 문구 풀) 변경 -- 이유: 헤드라인 표시 방식과 무관한 별개 로직, 범위 밖으로 명시 합의됨.
- `dhb-eye`("살까? 말까?" 라벨), 신호등 등급 로직 변경 -- 이유: 이번 문제와 무관.
- 헤드라인 자리를 다시 짧은 배지+본문 구조로 되돌리는 것(질문에서 기각된 옵션) -- 이유: 사용자가 "카드 확장형 줄바꿈"을 명시적으로 선택함.
