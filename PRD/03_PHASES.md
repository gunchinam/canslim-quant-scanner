# 판단 포스터 한줄평 헤드라인 레이아웃 재설계 -- Phase 분리 계획

> 모바일과 데스크톱은 사용자가 "각각 다르게 최적화"를 선택했으므로 별도 Phase로 분리한다.

---

## Phase 1: 모바일 `dp-verdict-poster` 레이아웃 재설계

### 목표
드로워 모바일 뷰에서 40자 한줄평도 18px 이상 폰트로, 2~3줄 이내 자연스러운 줄바꿈으로 읽힌다.

### 기능
- [ ] `.dp-verdict-poster` 카드 세로 여백/구조를 문장 길이에 맞게 확장(현재 `padding:22px 24px`, `flex` 가로 배치 -- `.dvp-main`이 좁아지는 구조 재검토)
- [ ] `.dvp-word` 폰트 하한을 18px로 상향(현재 `clamp(20px,7vw,38px)` + JS 인라인으로 최소 20px까지만 내려가던 걸 재설계 후 실질 하한 18px 보장)
- [ ] `app.js`의 `_dvpWordPx` 계산식(`app.js:3755`)을 새 레이아웃에 맞게 재조정하거나, 레이아웃이 충분히 넉넉해지면 JS 폰트 축소 로직 자체를 단순화/제거 검토
- [ ] 우측 상단 확신도 배지(`.dvp-conf`), 하단 워터마크(`.dvp-bg`)와 겹치지 않는지 확인

### 데이터
- `d.OneLiner`(기존 필드, 변경 없음)

### "진짜 제품" 체크리스트
- [ ] 실제 Flask 서버 기동 후 브라우저(또는 스크린샷)로 짧은/긴 한줄평 두 사례 모두 확인
- [ ] 배경색/확신도%/이유 설명 기존 그대로 동작
- [ ] 레이아웃 붕괴·텍스트 잘림·가로 스크롤 없음

### Phase 1 시작 프롬프트
```
이 PRD를 읽고 Phase 1을 구현해주세요.
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md
@PRD/04_PROJECT_SPEC.md

Phase 1 범위:
- web_app/static/scanner.css의 .dp-verdict-poster / .dvp-word 등 모바일 판단 포스터 레이아웃 재설계
- web_app/static/app.js의 _dvpWordPx 계산 로직 재조정

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- _pgCls/conv/_pvReason 로직 변경 금지
```

## Phase 2: 데스크톱 `dp-hero-banner-dt` 레이아웃 재설계

### 전제 조건
- Phase 1이 스크린샷으로 검증된 상태

### 목표
데스크톱 히어로 배너에서 40자 한줄평도 모바일과 같거나 더 큰 최소 폰트로, 자연스럽게 읽힌다.

### 기능
- [ ] `.dhb-verdict`(현재 `min-width:90px`, `.dhb-reason`과 좌우로 나란히 배치되는 좁은 고정폭 컬럼)이 병목이므로, 데스크톱 화면 폭을 더 활용하는 구조로 재설계(예: 헤드라인 컬럼 폭 확장, 또는 헤드라인과 이유를 세로로 재배치하는 등 모바일과 다른 구조 검토)
- [ ] `.dhb-word` 폰트 하한을 모바일(18px) 이상으로 상향(현재 `clamp(16px,2.4vw,26px)`)
- [ ] `app.js`의 `_dhbWordPx` 계산식(`app.js:3756`) 재조정

### 추가 데이터
- 없음(Phase 1과 동일하게 `d.OneLiner` 재사용)

### 통합 테스트
- Phase 1(모바일) 레이아웃이 여전히 정상 동작하는지 확인

## Phase 3: 회귀 검증 + 표본 스크린샷 검수

### 전제 조건
- Phase 1 + 2 구현 완료

### 목표
레이아웃 변경이 다른 기능을 깨뜨리지 않았음을 확인하고 골을 종료한다.

### 기능
- [ ] `tests/test_one_liner_consistency.py` 재실행, 기존 베이스라인(2 failed/24 passed) 유지 확인
- [ ] `web_app/tests/`(`test_history_timeline.py` 제외) 재실행, 195 passed 베이스라인 유지 확인
- [ ] 짧은/긴 한줄평 여러 종목으로 모바일+데스크톱 스크린샷 재검증
