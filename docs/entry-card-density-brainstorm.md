# 진입 타이밍 카드 — 정보 과밀 개선 브레인스토밍

> 발단: "지금 사진을 보면 너무 많은 정보를 한번에 꽉꽉 담으려고 한 느낌이야"
> 대상: `web_app/static/app.js` `_renderEntryCard()` / `#dp-entry-card` (스캐너 슬라이드 패널)
> 날짜: 2026-05-18

## 1. 문제 정의 (Opportunity)

**누가**: 스캐너에서 종목을 클릭해 진입 타이밍을 빠르게 판단하려는 개인 투자자.
**무엇을 원하나**: "지금 사야 하나 / 기다려야 하나 / 거르나" 한 가지 결정. 그 결정에 필요한 숫자 2~3개.
**현재 카드가 한 화면에 담는 것 (스크린샷 기준)**:

1. "진입 타이밍" 헤더 + 점수 `42 / 100`
2. 한줄 문구 `관망 · VWAP 과확장 · 강한 정배열`
3. 빨강→초록 그라데이션 바
4. 칩 2개 `VWAP 과확장` `강한 정배열`
5. 가격 그리드 4칸: 진입가(풀백) 251.14 / -7.35%, 손절 244.18, 목표가1(T1) 311.41, 목표가2 345.5
6. `유형 NEUTRAL · 풀백 대기`
7. `손익비 R:R 8.7:1 (현재 1.5)`
8. `손절기준 지지선 · 승률 19%`, 현재 271.07
9. `▼ 상세 분석` 확장

**핵심 진단**
- **중복**: ②의 문구가 ④의 칩과 동일 정보. ⑥의 "관망/풀백"이 ②의 "관망"과 중복.
- **위계 부재**: 진입가·손절·목표가1·목표가2가 모두 같은 시각 비중 → "지금 뭘 하라는 건지" 안 보임.
- **해석 없는 원자료**: `R:R 8.7:1 (현재 1.5)`, `승률 19%` 는 숫자만 던지고 의미를 사용자에게 떠넘김.
- **상충 신호 동시 노출**: 점수 42(낮음) + "강한 정배열"(강세) + 승률 19%(약함) 가 한 번에 → 인지 부하.
- **점진적 공개 1단**: `▼ 상세 분석` 하나뿐, 그 위는 전부 최대 밀도.

## 2. 세 관점 아이디어 (각 5개)

### A. Product Manager (가치·차별화)

1. **단일 결정 헤드라인**: 카드 최상단을 점수가 아니라 한 단어 액션으로 — `지금 매수` / `풀백 대기` / `관망` / `회피`. 점수·근거는 그 아래 부속.
2. **"왜 이 결론인가" 1줄 근거만**: 칩·문구·유형을 1개 문장으로 통합 ("정배열은 강하나 VWAP에서 7% 과확장 → 풀백까지 대기"). 중복 3덩어리 제거.
3. **신뢰도 뱃지**: 승률 19%·R:R 괴리(8.7 vs 현재 1.5)를 숫자 대신 `신뢰도 낮음` 한 뱃지로 추상화, 탭하면 숫자.
4. **결정에 안 쓰이는 필드 강등**: 목표가2, 손절기준(지지선), 현재가 중복은 기본 접힘. 상위 카드엔 진입·손절·목표가1만.
5. **시장국면 연동 톤다운**: 하락장(BearCap)일 땐 공격 지표(목표가2·R:R)를 더 숨기고 "리스크 우선" 레이아웃으로 자동 전환.

### B. Product Designer (UX·인지 부하)

1. **3존 레이아웃**: (1) 결론존 — 액션 1개 + 점수, (2) 행동존 — 진입/손절/목표가1 3숫자만, (3) 더보기존 — 나머지 전부 접기. 스크린샷의 9덩어리 → 3존.
2. **시각 위계 강제**: 진입가를 가장 크게(현재 -7.35% 디스카운트 강조), 손절은 빨강 보조, 목표가1은 초록 보조, 그 외 11px 회색.
3. **그라데이션 바 → 단일 마커**: 빨강~초록 바 위 점수 위치만 1개 점/눈금으로. 색면적 줄여 시선 분산 감소.
4. **칩 상한 2개 + 중복 제거**: 한줄 문구를 없애고 칩만 남기거나, 칩을 없애고 문구만. 둘 중 하나로 통일.
5. **점진적 공개 2단**: `▼ 상세 분석` 안에 또 묶지 말고, 카드 자체에 "요약 / 전체" 토글. 기본=요약(3존), 토글 시 현재의 풀 정보.

### C. Software Engineer (구현·플랫폼)

1. **밀도 모드 상태값**: `localStorage` `entryCardDensity = compact|full`. `_renderEntryCard`에서 compact면 px-t2·stop-method·breakdown·aq-row DOM 미생성(렌더 비용↓ + 시각 정리 동시 해결).
2. **표현 로직 서버화**: `_renderEntryCard`가 raw 필드를 다 그리는 대신, 백엔드 `_analyze_ticker`/`entry_plan`에서 `headline_action`, `confidence_band`, `one_reason` 3개 파생 필드를 만들어 내려주고 프론트는 그대로 표시 (프론트 분기 제거).
3. **`score_breakdown` 지연 로드**: dp-entry-breakdown·dp-aq-reasons는 "상세 분석" 펼칠 때만 생성 (현재는 항상 join/innerHTML).
4. **임계 기반 필드 가시성 규칙 테이블**: `{field: showWhen(plan)}` 규칙 맵으로 "목표가2는 R:R>3 & 정배열일 때만" 류를 데이터 주도로 — UI 코드에서 if 산재 제거, A11y/테스트 용이.
5. **카드 변형 A/B 훅**: `_renderEntryCard(d, variant)` 시그니처 추가 + variant를 쿼리/플래그로. compact vs 현행을 같은 데이터로 토글해 사용자 피드백 수집.

## 3. 우선순위 Top 5 (전 관점 통합)

신규/검증 관점 가중: ① 핵심 가치(결정을 빠르게) ② 빠른 검증 ③ 차별화.

| # | 아이디어 | 출처 | 왜 우선 | 검증할 핵심 가정 |
|---|---|---|---|---|
| 1 | **3존 레이아웃 (결론/행동/더보기)** | D1 | 과밀의 근본 원인=위계 부재. 가장 큰 체감 개선, 데이터 변경 없음 | "사용자는 진입·손절·목표가1 3개면 결정 가능, 나머지는 on-demand로 충분하다" |
| 2 | **단일 결정 헤드라인 (액션 1단어)** | A1 | 점수 42 같은 추상수치보다 "풀백 대기"가 즉시 행동가능. 차별화 큼 | "한 단어 액션이 점수 게이지보다 의사결정 속도·만족도를 높인다" |
| 3 | **중복 제거: 문구 ↔ 칩 ↔ 유형 통합** | A2/D4 | 즉시 적용 가능(저위험), 스크린샷의 3중 중복 바로 해소 | "②④⑥은 같은 정보 — 하나로 합쳐도 정보 손실이 없다" |
| 4 | **승률·R:R 괴리 → 신뢰도 밴드 추상화** | A3 | 가장 해석 어려운 원자료를 1뱃지로. 인지 부하 급감 | "사용자는 19%·8.7:1(1.5) 원수치보다 '신뢰 낮음'을 더 정확히 해석한다" |
| 5 | **밀도 모드 토글 (compact/full) + 상태 저장** | C1/D5 | 1·3을 끄지 않고 검증할 안전장치(고급 사용자 풀뷰 유지). A/B 기반 마련 | "기본 compact로 줘도 풀뷰 복귀율이 낮다(=compact가 다수에게 충분)" |

### 적용 순서 제안
- **Spike (검증용, 빠름)**: #3 중복 제거 → #1 3존 재배치. 데이터/백엔드 변경 0, `_renderEntryCard` 내 마크업·CSS만. 같은 종목으로 before/after 스크린샷 비교.
- **그다음**: #2 헤드라인 (백엔드 파생필드 `headline_action` 필요 → C2와 함께), #4 신뢰도 밴드.
- **마지막**: #5 토글로 고급 사용자 안전망 + 향후 A/B.

## 4. 비검증 시 리스크 / 반증 신호
- #1·#3로 숨긴 필드를 "상세 분석"에서 자주 다시 펼치면 → 그 필드는 핵심이었다는 신호, 3존 경계 재조정.
- #2 헤드라인이 오판을 부르면(예: "지금 매수" 후 손실) → 액션 문구는 보수적으로(관망/대기 우선).
- #4 밴드가 너무 뭉뚱그려져 고급 사용자가 불신 → 탭-투-리빌 원수치 항상 1탭 거리 유지.
