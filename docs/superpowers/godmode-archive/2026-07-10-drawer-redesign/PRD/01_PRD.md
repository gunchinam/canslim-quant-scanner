# 종목 상세 드로워 재설계 — PRD (Product Requirements Document)

> 생성일: 2026-07-10
> 생성 도구: Show Me The PRD

---

## 1. 제품 개요

### 한 줄 요약
종목 스캐너 웹앱의 종목 상세 드로워(`web_app/templates/detail.html`)를, 정보는 그대로 두되 "결론(점수·등급)"과 "근거(가격·RSI·목표가·컨센서스)"가 한눈에 구분되도록 정보 구조를 재설계해 더 보기 편하게 만든다.

### 해결하는 문제
현재 드로워는 히어로 카드 안에 점수·등급·RSI·현재가·목표가·컨센서스가 시각적 위계 없이 함께 쌓여 있고, 실적/컨센서스/투자자동향 카드들도 각자 흰 배경 카드로 따로 떠 있어 "결론"과 "근거"가 구분되지 않는다. 탭도 활성 상태가 눈에 잘 띄지 않는다. 사용자는 이를 "전반적으로 다 애매하게 아쉬움"으로 표현했다 — 특정 버그가 아니라 전체적인 정보 위계·흐름의 문제.

### 핵심 가치
- 기존 2026-07-08 2차 라운드 플랜에서 정의된 브랜드 톤(`--brand-soft` 틴트, `--surface-subtle` 섹션 밴드)은 유지 — 이미 검증된 색 언어를 재사용.
- 이번 PRD의 차별점은 색 추가가 아니라 **정보 구조 자체의 재배치**: 결론군/근거군/보조군/탭군으로 명확히 분리.

## 2. 사용자

### 주요 사용자
- **누구**: 종목 스캐너 앱을 실사용하는 개인 투자자(제작자 본인 포함, 소수 사용자).
- **상황**: 스캐너 목록에서 종목을 탭/클릭해 상세 드로워를 열고, 짧은 시간 안에 "이 종목이 좋은지, 지금 가격이 싼지"를 판단하려는 상황.
- **목표**: 드로워를 열자마자 결론(점수·등급)을 먼저 파악하고, 그 아래에서 근거(가격·목표가·컨센서스)를 훑은 뒤, 필요하면 탭으로 들어가 세부 지표를 본다.

### 사용자 시나리오
1. 사용자가 스캐너 목록에서 종목 카드를 클릭하면
2. 드로워가 열리며 히어로 카드가 "결론(점수·등급)"을 가장 먼저 보여주고
3. 그 아래 근거 영역(가격·RSI·목표가·컨센서스)과 보조 카드(실적·투자자동향)가 섹션으로 구분되어 흐름처럼 이어지며
4. 필요 시 탭(CAN SLIM/기술/재무/공시뉴스/US인사이트/노무라식)을 눌러 상세 지표로 들어간다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
스캐너 목록에서 종목 클릭 -> 드로워 오픈 -> 결론(히어로) 확인 -> 근거(가격·목표가·컨센서스) 스캔 -> 필요 시 탭 진입
```

### 상세 흐름
1. **드로워 오픈**: 종목 카드를 클릭하면 드로워가 우측/전체 화면으로 슬라이드 인.
2. **결론 확인**: 히어로 카드 상단에서 점수·등급·한줄평을 즉시 파악.
3. **근거 스캔**: 히어로 카드 하단(현재가·RSI·목표가) → 실적/컨센서스/투자자동향 카드까지 섹션 구분된 흐름으로 훑는다.
4. **탭 진입(선택)**: 더 깊은 지표가 필요하면 탭 네비게이션에서 원하는 탭을 선택.

## 5. 성공 기준

- [ ] 히어로 카드 안에서 "결론(점수·등급·한줄평)"과 "근거(현재가·RSI·목표가)"가 배경색/여백/타이포그래피로 시각적으로 구분된다.
- [ ] 히어로/실적/컨센서스/투자자동향 카드가 개별 흰 카드로 따로 떠 보이지 않고, 섹션 라벨 밴드로 하나의 흐름처럼 연결되어 보인다.
- [ ] 탭 네비게이션에서 활성 탭이 비활성 탭과 명확히 구분된다(색상·굵기·밑줄 등).
- [ ] 목표가 박스와 컨센서스 박스가 `--brand-soft` 톤으로 강조되어 "결론 지점"으로 인지된다.
- [ ] 정보값 없는 히어로 장식 문구("🏆 이 회사 자체는 좋은가?" 라벨, hero-grade-dot, hero-watermark), 점수 추세 스파크라인, 실적/컨센서스 카드의 중복 보조 안내 문구(재무 지표 탭 안내, cs-summary)가 마크업에서 제거된다.
- [ ] 데스크톱(768px 초과)과 모바일(480px 기준) 두 뷰포트 스크린샷에서 레이아웃 깨짐(텍스트 겹침, 카드 잘림, 가로 스크롤 발생)이 없다.
- [ ] 기존 pytest 회귀 테스트가 변경 전과 동일한 통과 수를 유지한다(기존 실패 10건은 이번 작업과 무관하므로 베이스라인으로 취급).

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- `.hero-oneliner`(16개 판정 태그별 halftone/줄무늬 텍스처 카드)와 `dp-verdict-poster` 컴포넌트 수정 -- 이유: 기존 2차 라운드 플랜에서도 명시적으로 범위 제외된, 이미 완성도 높은 컴포넌트.
- CAN SLIM 리스트, 소유구조/내부자 카드, 재무 표 등 데이터 나열 카드의 배경색·구조 변경 -- 이유: 흰 배경 데이터 카드는 그대로 유지하는 것이 기존 톤 결정.
- `web_app/templates/scanner.html` 등 드로워 외 다른 화면 수정 -- 이유: 이번 PRD는 `detail.html` 단일 파일 범위로 한정(Phase 3 후보로만 기록).
- 새 데이터 필드·API 추가, 백엔드 로직 변경 -- 이유: 기존 데이터를 재배치하는 순수 UI 작업.
- 모바일 세밀 튜닝, 탭 전환 애니메이션 -- 이유: Phase 2로 이연.
- 새 CSS 프레임워크/빌드 도구 도입, `theme.css` 신규 토큰 추가 -- 이유: 기존 인라인 CSS + Jinja2 스택과 기존 토큰(`--brand-soft`, `--surface-subtle`)만 재사용.
