## Phase 1: MVP (이번 골 범위)

### 목표
드로워를 열었을 때 결론(점수·등급)과 근거(가격·목표가·컨센서스)가 시각적으로 구분되고, 카드들이 하나의 흐름처럼 이어져 보이며, 탭 활성 상태가 명확해진다.

### 기능
- [ ] 히어로 카드 정보 위계 재배치(결론군/근거군 시각적 구분)
- [ ] 카드 간 섹션 구분(히어로/실적/컨센서스/투자자동향을 흐름처럼 정리)
- [ ] 탭 네비게이션 가독성 개선(활성 탭 강조)
- [ ] 목표가/컨센서스 박스 강조 승격(`--brand-soft` 톤 적용)

### 데이터
- 신규 없음. `02_DATA_MODEL.md`의 결론군/근거군/보조군/탭군 필드를 그대로 재사용.

### 인증
- 해당 없음(로그인/인증과 무관한 UI 작업).

### "진짜 제품" 체크리스트
- [ ] 실제 Flask 개발 서버로 렌더링 확인(정적 목업 X)
- [ ] 실제 종목 데이터(스캐너에서 클릭한 실제 종목)로 스크린샷 검증
- [ ] 데스크톱 + 모바일(480px) 두 뷰포트 모두 확인
- [ ] 기존 pytest 회귀 테스트 통과 수 유지

### Phase 1 시작 프롬프트
```
이 PRD를 읽고 Phase 1을 구현해주세요.
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md

Phase 1 범위:
- 히어로 카드 정보 위계 재배치(결론/근거 구분)
- 카드 간 섹션 구분
- 탭 네비게이션 가독성 개선
- 목표가/컨센서스 박스 강조 승격

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- web_app/templates/detail.html 한 파일만 수정
- .hero-oneliner, dp-verdict-poster, 데이터 나열 카드(CAN SLIM 리스트 등)는 건드리지 않음
- 새 데이터/API 추가 없이 기존 필드만 재사용
```

## Phase 2: 확장 (다음 라운드 후보)

### 전제 조건
- Phase 1이 병합되어 실제 화면에 반영된 상태.

### 목표
Phase 1에서 잡은 정보 구조를 모바일에서 더 세밀하게 다듬고, 탭 전환에 인터랙션 디테일을 더한다.

### 기능
- [ ] 모바일(768px 이하) 세밀 튜닝
- [ ] 탭 전환 애니메이션/인터랙션 디테일

### 추가 데이터
- 없음.

### 통합 테스트
- Phase 1 기능(결론/근거 구분, 섹션 밴드, 탭 강조, 목표가 승격)이 여전히 정상 동작하는지 확인.

---

## Phase 3: 고도화 (마지막, 별도 요청 시)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중이며 드로워에서 토큰/패턴이 검증된 상태.

### 목표
드로워에서 검증된 색 토큰·섹션 밴드 패턴을 `scanner.html` 등 다른 화면으로 확산한다.

### 기능
- [ ] 검증된 토큰/패턴을 다른 화면으로 확산(별도 작업 요청 시에만 착수)

### 주의사항
- 다른 화면(스캐너 목록 등)까지 건드리므로 회귀 테스트 범위가 넓어짐 — 반드시 별도 PRD/골로 분리해 진행.
