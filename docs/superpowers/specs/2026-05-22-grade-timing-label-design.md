# 종목 등급 · 진입 타이밍 라벨 분리 설계

**작성일:** 2026-05-22
**대상 파일:** `web_app/static/app.js`, `web_app/templates/scanner.html`

## 배경

리스트에서 "강력매수" 필터를 누르면 종목이 잡히는데, 그 종목 세부 페이지에서는 "진입 보류"가 표시된다. 사용자는 이 둘이 모순이라고 느낀다.

실제로는 모순이 아니다. 두 가지 서로 다른 축이다:

- **종목 퀄리티 축** — 시그널 `STRONG_BUY` → "강력 매수" (`_KO_MAP`). EPS·ROE·PER·모멘텀 등 종합점수. "좋은 회사인가?"
- **진입 타이밍 축** — 시그널 `HOLD` → "진입 보류" (`_KO_MAP`), 또는 `_compute_entry_status`의 EntryStatus. "지금 이 가격에 들어갈 타이밍인가?"

문제는 두 축 모두 "사라/사지마라" 명령형 어휘를 써서, 한 종목에서 라벨끼리 충돌하는 것처럼 읽힌다는 점이다.

## 목표

- 종목 퀄리티 축과 진입 타이밍 축을 서로 다른 어휘 체계로 분리해, 명령형끼리 충돌하지 않게 한다.
- 리스트에서 클릭 전에 두 축이 한눈에 보이게 한다.
- 백엔드 스코어링 로직·버킷 구조·EntryStatus 계산은 변경하지 않는다 (표시 계층만 수정).

## 비목표

- 종합점수, 버킷(`_raw_bucket`), EntryStatus 계산식 변경 — 이번 범위 밖.
- 세부 페이지 2축 사분면 카드(`_renderQuadrant`)의 구조 변경 — 어휘 통일만.

## 설계

### 1. 종목 등급 배지 (S/A/B/C)

리스트 시그널 컬럼의 명령형 라벨(강력매수/강력리더/매집/회피)을 종합점수 파생 등급으로 대체한다.

등급 컷 (사분면 '좋은 회사' 임계 60과 정합):

| 등급 | 종합점수(TotalScore) 조건 |
|---|---|
| S | ≥ 75 |
| A | 60 ~ 74 |
| B | 45 ~ 59 |
| C | < 45 |

- 등급은 `TotalScore` 파생값이므로 매 스캔마다 유동적으로 갱신된다.
- `TotalScore`가 없거나 숫자가 아니면 등급을 표시하지 않고 기존 시그널 라벨로 폴백한다.
- `app.js`에 순수 함수 `_stockGrade(totalScore)` 추가 — 점수를 받아 `'S'|'A'|'B'|'C'|null` 반환.

특수 버킷 처리:
- STORY_STOCK 버킷 종목은 등급 척도에 맞지 않으므로 등급 대신 "스토리" 칩으로 별도 표기한다.
- 판별: 종목 데이터의 버킷/시그널 필드가 `STORY_STOCK`이면 등급 계산을 건너뛰고 "스토리" 칩을 표시한다.

### 2. 진입 타이밍 배지

리스트의 기존 작은 신호등 점(`_entryLight`)을 라벨 달린 배지로 승격한다.

| EntryStatus | 배지 라벨 | 아이콘 |
|---|---|---|
| STRONG / GREEN | 진입적기 | 🟢 |
| NEUTRAL / YELLOW | 눌림대기 | 🟡 |
| AVOID / RED | 부적합 | 🔴 |

- `EntryStatus`가 없으면 배지를 표시하지 않는다 (현재 `_entryLight` 동작 유지).
- `_entryLight(stock)` 함수를 수정해 아이콘만 있던 것을 "아이콘 + 라벨" 배지로 변경.
- 결과적으로 리스트 한 행에서 "A등급 · 눌림대기"처럼 두 축이 클릭 전에 모두 보인다.

### 3. 필터 칩 정리

- 기존 "강력매수" 칩(`_signalTier(s.Signal) === 'strong'` 필터) → 라벨을 "S·A등급"으로 리네이밍. 필터 로직은 그대로 두되, 칩 텍스트만 변경.
- 기존 "진입적기" 칩(`entry_green` 필터, `EntryStatus === 'STRONG' || 'GREEN'`) → 유지.
- 두 칩이 서로 다른 축임이 이름만으로 드러난다.

### 4. 세부 페이지 어휘 통일

- `_renderQuadrant`의 사분면 라벨("좋은 회사 · 나쁜 타이밍" 등)은 구조 유지.
- 진입 타이밍 카드(`_renderEntryCard`)의 라벨이 리스트 배지(진입적기/눌림대기/부적합)와 같은 어휘를 쓰도록 통일.
- 백엔드 `_compute_entry_status`의 label("진입 강함"/"관망"/"진입 부적합")은 변경하지 않고, 프론트 표시 시 새 어휘로 매핑한다.

## 데이터 흐름

```
스캔 결과(stock) → 리스트 행 렌더
    ├─ TotalScore  → _stockGrade()  → S/A/B/C 등급 배지
    │                 (STORY_STOCK이면 "스토리" 칩)
    └─ EntryStatus → _entryLight()  → 진입적기/눌림대기/부적합 배지
                                       (아이콘 + 라벨)
세부 페이지 → _renderQuadrant (2축 사분면, 어휘 통일)
            → _renderEntryCard (진입 타이밍, 어휘 통일)
```

## 엣지케이스

| 케이스 | 처리 |
|---|---|
| TotalScore 결측/비숫자 | 등급 미표시, 기존 시그널 라벨로 폴백 |
| EntryStatus 결측 | 진입 배지 미표시 (기존 동작) |
| STORY_STOCK 버킷 | 등급 대신 "스토리" 칩 |
| 등급 컷 경계값 (예: 정확히 60) | 60은 A등급 (조건 `>= 60`) |
| 모바일 좁은 화면 | 배지 2개가 줄바꿈되거나 컴팩트 표시되도록 CSS 반응형 |

## 테스트

- **`_stockGrade()` 단위 테스트:** 75→S, 74→A, 60→A, 59→B, 45→B, 44→C, 결측→null.
- **STORY_STOCK 종목:** 등급 배지 대신 "스토리" 칩 표시 확인.
- **진입 배지:** STRONG→진입적기🟢, NEUTRAL→눌림대기🟡, AVOID→부적합🔴, 결측→배지 없음.
- **혼동 시나리오 재현:** S등급이면서 진입 타이밍 눌림대기인 종목이 리스트에서 "S등급 · 눌림대기"로 함께 표시되는지 확인 — 클릭 후 세부 페이지와 모순 없이 일관.
- **필터 칩:** "S·A등급" 칩과 "진입적기" 칩이 각각 다른 종목 집합을 거르는지 확인.
