# Hero Zone 밀도 리디자인 — 설계 문서

**날짜:** 2026-06-09
**상태:** 승인 대기 (사용자 리뷰 전)
**대상:** 종목 상세 드로워(detail panel)의 Hero Zone
**브랜치:** main

---

## 1. 문제 정의

직전 세션에서 Hero Zone을 Zone A/B/C 단일 컬럼 "판결문" 레이아웃으로 리디자인했다. 모바일에서는 깔끔하지만, **데스크탑(패널 폭 `min(980px, 100%)`)에서 단일 컬럼이 넓은 폭에 얇게 퍼지면서 빈 공간이 과도하게 느껴진다("휑하다").**

사용자 진단(비주얼 컴패니언 세션):
- **빈 공간 ①** — 92점 점수 우측 절반이 거의 공백
- **빈 공간 ②** — 현재가·등락률 셀의 값 아래 공백
- 문제의 본질: **밀도(공간이 빔) + 콘텐츠(채울 의미 있는 데이터 부족) 둘 다**
- **한줄평(one-liner)이 이 화면의 키포인트** → 더 키워야 하며 절대 축소 금지

## 2. 목표 / 비목표

**목표**
- 데스크탑 폭에서 Hero Zone이 "꽉 찬" 느낌이 들도록 가로 폭을 의미 있게 사용
- 한줄평 포스터를 화면의 압도적 주인공으로 강화
- 표(table) 느낌 없이 프리미엄(토스/로빈후드) 톤 유지
- 모바일은 기존 단일 컬럼 흐름 유지

**비목표**
- 차트 탭의 4축 핸드드로잉 차트 대체 (별개)
- 매수/매도 텍스트 도입 (금지 — 기존 제약 유지)
- 점수/시그널 산출 로직 변경 (UI만)

## 3. 확정 디자인 — "B1 + 한줄평 강화"

비주얼 컴패니언에서 여러 방향(A 폭제한 / B 차트동반 / C 타일 / D1~3 / E1~3)을 거쳐 **B1(차트-포워드 프리미엄)** 으로 수렴했고, 한줄평을 주인공으로 복귀·확대했다.

### 레이아웃 구조 (데스크탑)

```
┌─────────────────────────────────────────────────────────┐
│  ① 한줄평 포스터 (주인공)                                 │
│     - 그라데이션 컬러 카드, 큰 따옴표 장식, 강한 그림자    │
│     - 텍스트 46px / weight 800 / line-height 1.25         │
│     - "한줄평" eyebrow 라벨 + "신호 정렬 · 강세 국면" 태그 │
├──────────────────────────┬──────────────────────────────┤
│  ② 좌: 판결 + 수치        │  ③ 우: 20일 추세 차트         │
│     92점(46px) · S 강세    │     · 영역 스파크라인 + 글로우 │
│     · Risk 48 주의 칩      │     · ▲12.4% 변화 배지        │
│     ─────────────────     │     · 마지막 값 플로팅(163.12) │
│     현재가 │ 등락률         │     ─────────────────────────│
│     RSI    │ RS등급        │     52주위치 바│거래량│시총   │
└──────────────────────────┴──────────────────────────────┘
```

### 한줄평 포스터 사양 (① — 키포인트)
- 배경: `linear-gradient(135deg, 시그널색 밝은톤, 시그널색 어두운톤)` — 시그널에 따라 색 동적 변경(강세=녹색 계열). 기존 `dp-oneliner-poster` 색 로직 재사용.
- 패딩: `50px 30px 44px`, radius 22px, `box-shadow: 0 16px 40px rgba(시그널색,.28)`
- 큰 장식 따옴표(`"`) 좌상단, opacity .12
- 텍스트: 46px / 800 / -0.035em / line-height 1.25 / 살짝 text-shadow
- 하단: 반투명 pill에 "신호 정렬 · 강세 국면" (기존 signal phase 텍스트 재사용)

### Zone A — 판결 (②-좌 상단)
- 점수 46px (한줄평이 1순위이므로 기존 60px → 46px로 양보), weight 800, 시그널 색
- "종합 등급" eyebrow + "S · 강세" (기존 `dp-signal`)
- Risk 칩(`dp-risk-gauge`) 우측 정렬, outline pill 스타일
- 주도주 배지(`dp-leader-badge`) 유지

### Zone B — 수치 (②-좌 하단, 2×2)
- 현재가 / 등락률 / RSI / RS등급 — 기존 ID 유지
- uppercase 마이크로 라벨 + tabular-nums

### Zone C — 추세 차트 패널 (③-우, 신규)
- **20일 영역 스파크라인** (closes 기반), 시그널 색 라인 + 그라데이션 fill
- 마지막 점 글로우 + 마지막 값 라벨
- "최근 20일" 라벨 + **▲N% 변화 배지**
- 하단 보조 3종: **52주 위치 바** / **거래량 배수** / **시총**

### 모바일 (≤768px)
- 2-컬럼 → 세로 스택: ① 한줄평 → ② 판결+수치 → ③ 차트
- 한줄평 텍스트 clamp로 축소(기존 동작), 차트 풀폭

## 4. 데이터 가용성 (실측)

| 데이터 | 출처 | 상태 |
|--------|------|------|
| 현재가 `d.Price` | scan payload | ✅ 있음 |
| 등락률 `d.DayChange` (`dp-day-chg`) | payload | ✅ 있음 |
| RSI / RS (`dp-rs-value`) | payload | ✅ 있음 |
| 거래량 배수 `d.VolRatio` | payload (app.js:3327) | ✅ 있음 |
| 시총 `d.MarketCap` / `_MarketCap` | payload (app.py:1738, app.js:1604) | ✅ 있음 |
| 52주 위치 % | `price_levels.target_52w_high` 등으로 계산 | ⚠ 계산 필요 (high/low 확보 시 `(price-low)/(high-low)`) |
| 20일 closes (스파크라인) | 4축 차트 엔드포인트가 이미 `hist` fetch (app.py:2375~) | ⚠ 경량 closes 배열을 상세 payload에 추가 필요 |

**핵심 작업 의존성:** 20일 스파크라인 = 서버가 `hist`에서 종가 ~20개를 추려 상세 응답(또는 별도 경량 필드)에 포함 → 클라이언트가 인라인 SVG로 렌더. 52주 위치는 high/low 가용 시 클라이언트 계산.
