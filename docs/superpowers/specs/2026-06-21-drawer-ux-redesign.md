# 드로워 UX 재설계 — 질문 기반 탭 구조

## 배경

현재 드로워는 정보가 풍부하지만 "살까 말까"와 "언제 살까"라는 핵심 질문의 답이 어디 있는지 한눈에 보이지 않는다. 탭 이름이 데이터 타입(결론/타이밍/회사/상세) 기준이어서 사용자가 어떤 탭을 열어야 할지 알기 어렵다.

## 목표

드로워를 열었을 때 3초 안에 "살까?"와 "언제?"에 대한 답이 보여야 한다.

## 탭 구조 변경

| 기존 | 변경 후 | 핵심 질문 |
|------|---------|-----------|
| 결론 | 살까? | 이 종목 살 만한가? |
| 타이밍 | 언제? | 지금이 타이밍인가? |
| 상세 | 얼마나? | 얼마나 살까? |
| 회사 | 뭔데? | 이 회사가 뭐하는 곳? |
| 참고 | (유지) | — |

## 각 탭 설계

### 살까? 탭

첫 줄에 큰 글씨로 한 단어 답:
- `사세요` (conv ≥ 72, 초록)
- `기다리세요` (conv 42~71, 노랑)
- `보류예요` (conv < 42, 빨강)

그 아래 이유 목록 (✅ 긍정 / ⚠️ 주의):
- 4축 분석에서 score ≥ 4인 항목 → ✅
- score ≤ 2 또는 특수 신호(bear_div) → ⚠️

기존 공포탐욕 게이지, Risk 배지, BF Score, 투자자 동향은 이 탭 하단에 축약 표시.

### 언제? 탭

첫 줄에 큰 글씨로 한 단어 답:
- `지금이에요` (stars 4~5)
- `조금 기다려요` (stars 3)
- `아직이에요` (stars 1~2)

그 아래 기존 4축 타이밍 분석 (phase, 이유 목록, 상세 접기/펴기).

### 얼마나? 탭

기존 분할매수 플랜 카드를 단독 배치:
- 1차 / 2차 / 3차 가격 및 비중
- 평균단가 · 최대 이격
- 손절 기준 (ATR 기반)

### 뭔데? 탭

기존 회사 탭 내용 유지:
- 사업 요약, 섹터, 시총
- F-Score, Altman Z
- 컨센서스 목표가

## 변경 범위

- `scanner.html`: 탭 버튼 텍스트 4개 변경
- `scanner.html`: 살까? 탭에 verdict-word 영역 추가
- `scanner.html`: 언제? 탭에 timing-word 영역 추가
- `scanner.html`: 얼마나? 탭 신설 (분할매수 플랜 이동)
- `app.js`: verdict-word 렌더 함수 추가 (conv → 한 단어)
- `app.js`: timing-word 렌더 함수 추가 (stars → 한 단어)
- `scanner.css`: verdict-word / timing-word 스타일 추가

## 변경하지 않는 것

- 백엔드 로직, 점수 계산 — 일절 변경 없음
- 기존 데이터 구조 및 API
- 참고 탭

## 성공 기준

드로워를 열었을 때 스크롤 없이 "살까?" 탭에서 "사세요 / 보류예요"가 보이고, "언제?" 탭에서 "지금이에요 / 아직이에요"가 보인다.
