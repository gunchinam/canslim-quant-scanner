# 종목스캐너 디자인 전면 개편 — HP 디자인 언어 기반

## 배경

현재 `web_app`은 애플 디자인 언어(`theme.css`) 기반으로 SF Pro 타이포그래피, 애플 블루(`#0071E3`) 브랜드 컬러, 18px 단일 radius, 5단계 그림자 체계, 라이트/다크 모드를 갖추고 있다.

사용자가 참고한 두 디자인 문서(`DESIGN-airbnb.md`, `DESIGN-hp.md`) 중 **HP 디자인 언어**를 기본 방향으로 채택한다. Airbnb의 대폭 둥근 pill 형태보다, HP의 "날카로운 버튼 + 부드러운 카드" 2단 radius 철학과 단일 블루 액센트가 데이터 밀도 높은 금융 대시보드에 더 잘 맞는다고 판단했다.

HP·Airbnb 문서 모두 소비자 마케팅/커머스 사이트를 위한 것이라, 히어로 배너나 프라이싱 카드 같은 컴포넌트는 그대로 이식하지 않는다. **색상·타이포그래피·radius·elevation·섹션 밴드 리듬 같은 "느낌"만 가져오고, 컴포넌트는 종목 테이블·카드·필터칩 등 이 앱의 실제 화면에 맞게 재해석**한다.

다크모드는 기존에도 실질적으로 사용되지 않았으므로 이번 리디자인 범위에서 제외한다. 기존 `.dark` 클래스는 건드리지 않고 그대로 둔다(유지보수 대상 아님).

## 범위

- 전체 4개 페이지 동시 적용: `scanner.html`(메인 스캐너), `detail.html`(종목 상세 드로어), `compare.html`(비교), `pyramid.html`(피라미딩 계산기)
- 대상 파일: `web_app/static/theme.css`(토큰), `web_app/static/scanner.css`(컴포넌트), 4개 템플릿의 인라인 스타일/클래스
- 변경 깊이: **토큰 재정의 + 핵심 컴포넌트 리스타일링 + 레이아웃 섹션 리듬**까지 포함하는 전면 재설계
- 다크모드(`.dark` 클래스)는 범위 밖 — 손대지 않음

## 색상

기존 `--brand`(애플 블루) 계열 토큰을 HP 팔레트로 교체하고, 주식 데이터 특유의 의미 색상(상승/하락, 등급)은 HP 문서에 없는 확장 토큰으로 별도 추가한다. 확장 토큰도 HP의 절제된 채도 원칙을 따른다.

| 용도 | 토큰 | 값 | 비고 |
|---|---|---|---|
| 브랜드 액센트 | `--brand` | `#024ad8` | HP Electric Blue. CTA·링크·활성 탭에만, 뷰당 최대 2곳 |
| 브랜드 프레스 | `--brand-pressed` | `#0e3191` | 버튼 pressed 상태 |
| 브랜드 연함 | `--brand-soft` | `#c9e0fc` | 선택 칩, 연한 배지 배경 |
| 캔버스 | `--background` | `#ffffff` | 페이지 기본 배경(기존과 동일) |
| 클라우드 | `--surface-subtle` | `#f7f7f7` | 통계바, 매크로 스트립 등 유틸리티 밴드 |
| 포그 | `--surface-muted` | `#e8e8e8` | 구분선 겸 배경, 강조 밴드 |
| 잉크(주 텍스트) | `--text-primary` | `#1a1a1a` | 기존 `#1D1D1F`에서 톤 통일 |
| 차콜(보조 텍스트) | `--text-secondary` | `#3d3d3d` | |
| 그래파이트(약한 텍스트) | `--text-tertiary` | `#636363` | |
| 잉크 슬랩 | `--ink-slab` | `#1a1a1a` | 종목 상세 드로어 핵심 요약 헤더, 공유 카드 배경 등 리듬을 닫는 용도로 제한적 사용 |
| 상승(확장) | `--stock-up` | `#0f8a5f` | HP엔 없는 신규 토큰. 절제된 그린 |
| 하락(확장) | `--stock-down` | `#b3262b` | HP `bloom-deep` 재사용 |
| 등급 S | `--grade-s` | `#7C3AED`(유지) | 브랜드 팔레트와 독립된 신호로 유지 |
| 등급 A | `--grade-a` | `#024ad8` | 기존 `#2563EB`에서 브랜드 블루로 통일 |
| 등급 B/C/D | `--grade-b/c/d` | 기존 값 유지, 채도만 소폭 하향 | 톤 통일 |

## 타이포그래피

Pretendard를 1차 폰트로 확정(이미 폴백으로 로드 중 — SF Pro 참조를 전부 정리). HP 원칙인 "디스플레이도 500~600 웨이트 중심, 700은 강조 전용"을 따른다.

| 토큰 | 크기 | 굵기 | 용도 |
|---|---|---|---|
| `display-lg` | 28px | 600 | 페이지 타이틀(대시보드 특성상 최소 사용) |
| `display-md` | 20px | 600 | 섹션 헤드 ("종목 상세" 등) |
| `title` | 16px | 600 | 카드 타이틀, 종목명 |
| `body` | 14px | 400 | 기본 테이블/카드 본문 |
| `body-emphasis` | 14px | 500 | 강조 셀(점수, 등급명) |
| `caption` | 12px | 400 | 메타 정보, 타임스탬프 |
| `button` | 14px | 600 | 한글 버튼엔 HP의 uppercase+letter-spacing 미적용 — 굵기로만 강조 |
| `price` | 16~18px | 500 | 가격/숫자 셀. `font-variant-numeric: tabular-nums` 고정 |

## 형태 (Radius)

HP의 2단 철학: **버튼·인풋은 날카롭게, 카드·모달은 부드럽게**. 기존 단일 `--radius: 18px` 토큰을 4개로 분리한다.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--radius-sharp` | 4px | 버튼, 인풋 |
| `--radius-pill` | 9999px | 필터 칩, 세그먼트 컨트롤(마켓 선택 등) |
| `--radius-card` | 16px | 종목 카드, 드로어, 모달, 통계 카드 |
| `--radius-badge` | 8px | 등급 배지, 뱃지 pill |

## 입체감 (Elevation)

그림자를 최소화하고 색 대비로 깊이를 표현한다. 기존 5단계 그림자 체계를 4단계로 단순화.

| 레벨 | 처리 | 용도 |
|---|---|---|
| 0 Flat | 그림자 없음 | 페이지 배경, 유틸리티 밴드 |
| 1 Hairline | 1px `#e8e8e8` 보더 | 테이블 셀, 아웃라인 버튼 |
| 2 Soft Lift | `0 2px 8px rgba(26,26,26,0.08)` | 종목 카드, 통계 카드, 드로어 |
| 3 Floating | `0 8px 24px rgba(26,26,26,0.12)` | 드롭다운, 모달, 토스트 |

카드 hover 시 그림자를 과하게 키우지 않는다(레벨 변경 없이 배경색 대비로만 반응).

## 레이아웃 리듬

HP의 캔버스 → 클라우드 → 포그 → 잉크 밴드 구조를 대시보드 섹션에 매핑한다.

- **캔버스(흰색)**: 상단바, 종목 테이블/카드 영역 — 메인 작업 공간
- **클라우드(`#f7f7f7`)**: 매크로 이벤트 스트립, 통계바(stats-bar)
- **포그(`#e8e8e8`)**: 디스클레이머 밴드, 활성 필터 칩 배경
- **잉크 슬랩(`#1a1a1a`)**: 종목 상세 드로어의 핵심 요약 헤더, 공유 카드 배경에 한정 — 리듬을 닫는 용도로만 최소 적용

## 주요 컴포넌트

**버튼**
- `button-primary`: `--brand` 배경, 흰 텍스트, `--radius-sharp`, 40px 높이 — "스캔", "저장" 등 주 액션
- `button-outline`: 흰 배경 + `--brand` 텍스트/보더 — "내보내기", "필터 리셋" 등 보조 액션
- `button-ink`: `--text-primary` 배경 필채움 — 잉크 슬랩 위 CTA용

**세그먼트 컨트롤 / 필터 칩**
- `segment-pill`: 마켓 선택(미국/한국). `--radius-pill`, active 시 잉크 배경 + 흰 텍스트(기존 `.btn-seg` 교체)
- `filter-chip`: 스캐너 필터 칩(전체/관심종목/S·A등급 등). 동일 pill 형태, active는 `--brand-soft` 배경 + `--brand` 텍스트로 세그먼트 컨트롤과 시각적 구분

**종목 카드 / 테이블 행**
- 카드형 뷰(모바일): `--radius-card` + Soft Lift. 헤더에 종목명(`title`)+등급 배지, 본문에 가격/점수 그리드
- 테이블형 뷰(데스크톱): 그림자 없이 Hairline 보더만. hover 시 `--surface-subtle` 배경으로만 반응(그림자 변화 없음)

**등급/신호 배지**
- `grade-badge`: `--radius-badge`, 필채움. 등급별 컬러는 색상 섹션 참조
- `signal-badge`(상승/하락/급등 등): `--stock-up`/`--stock-down` 텍스트 + 10% 투명도 배경 — HP의 절제된 배지 스타일

**인풋**
- `text-input`(검색창 등): `--radius-sharp`, 44px 높이, 기본 `#c2c2c2` 보더 → 포커스 시 `--text-primary` 보더로 전환(halo 없음)

## 다크모드

범위 제외. 기존 `.dark` 클래스 스타일은 건드리지 않고 그대로 둔다. 라이트 모드 토큰 변경과 별개로 유지되며, 시간이 지나며 라이트 모드와 다소 어긋나더라도 감수한다(사용되지 않는 기능이므로).

## 반응형

기존 브레이크포인트(모바일/태블릿/데스크톱) 구조와 카드/테이블 전환 로직은 그대로 유지한다. 이번 개편은 색상·타이포·radius·elevation 토큰 교체 + 컴포넌트 리스타일링이 핵심이며, 그리드/브레이크포인트 자체는 변경하지 않으므로 레이아웃 붕괴 위험은 낮다.

## 알려진 제약

- HP 원본 문서의 hover 색상, 로딩/스켈레톤 상태는 문서에 없어 이번 스펙에도 포함하지 않는다 — 구현 단계에서 기존 hover 로직을 색 토큰만 교체해 재사용한다.
- Grade B/C/D의 정확한 신규 색상값은 구현 단계에서 기존 값 기준으로 채도만 조정하며 확정한다(이 스펙에서는 방향성만 제시).
- `compare.html`, `pyramid.html`은 `scanner.html`만큼 상세히 훑지 않았으므로, 구현 계획 단계에서 두 페이지의 기존 컴포넌트를 다시 확인하고 위 토큰 체계에 맞춰 매핑한다.
