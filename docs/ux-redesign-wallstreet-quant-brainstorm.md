# UX/UI 개선 브레인스토밍 — 월가 퀀트 패널 관점

**작성일:** 2026-05-25
**대상:** 종목스캐너 (Flask 웹앱, scanner.html / multibagger.html / detail.html)
**주 사용자:** 한국 개인투자자, 미국·한국 종목 스캔
**관점:** Goldman Sachs · BlackRock · Morgan Stanley · Renaissance · Two Sigma · Citadel 퀀트 트레이더/엔지니어

---

## 0. 컨텍스트 진단

현 UI(scanner.html 3,828줄 · app.js 4,935줄)는 **단일 거대 페이지에 카드/테이블/한줄평·통계를 동시에 노출**하는 "리테일 스캐너" 패턴이다. 월가 데스크 관점에서 보면:

- **신호-잡음비 낮음**: 모든 종목이 동일 무게로 표시 → 트레이더는 즉시 "오늘 무엇이 변했나"를 못 봄
- **확신도(confidence) 누락**: 점수만 있고 시그널의 통계적 신뢰도/표본 크기가 카드에 안 보임
- **리스크 컨텍스트 부재**: VIX·시장 레짐·섹터 베타가 종목 옆에 없음 → 사이징 판단 불가
- **시간 차원 누락**: "어제 대비 무엇이 새로 떴는가(deltas)"가 시각적으로 드러나지 않음

## 1. 세 관점 × 5 아이디어

### A. PM (Product Manager) — 시장 적합성·가치·차별화

**A1. Regime-Aware Top Bar (Bridgewater × Citadel)**
- 상단에 항상 보이는 4-칸 레짐 인디케이터: `VIX 레벨/Δ` · `시장 폭(Advancers/Decliners)` · `섹터 회전 1위` · `유동성 상태`
- 코드에 이미 있는 VIX-스케일 sizing(circuit breaker) 신호를 UI 1급 시민으로 승격

**A2. "What Changed Today" 우선 뷰 (Renaissance: 변화 추적)**
- 기본 정렬을 "점수 절댓값"이 아닌 **"지난 5일 대비 점수/모멘텀 델타"**로
- "신규 진입(↑신호)" / "이탈(↓신호)" / "유지" 3 탭. 매일 처음 열 때 가장 큰 신호.

**A3. Confidence 배지 (Two Sigma: 통계적 엄밀성)**
- 모든 한줄평·점수 옆에 **신뢰 등급**: ●●● (표본 충분+백테스트 검증) · ●● (rule-based) · ● (휴리스틱)
- "이 시그널은 백테스트 N=240 케이스에서 적중률 62%" 같은 클릭형 툴팁

**A4. Strategy Books — 미리 묶은 시나리오 (Goldman Sachs prime brokerage 발상)**
- "AI Capex 1차 수혜" / "금리 인하 베네피셔리" / "방어주 회전" 같은 **테마 북** 구조
- 사용자가 매번 필터를 조합하는 대신, PM이 큐레이팅한 합본을 클릭 한 번에 적용

**A5. Position Sizing Helper (Kelly × Citadel risk budgeting)**
- 종목 카드 우상단에 "이 종목에 포트의 X% 권장" — VIX·베타·점수·자기 자본 입력 기반
- 단순 추천이 아니라 "왜 X%인가"의 1줄 근거 표시

### B. Designer — UX·온보딩·인게이지먼트

**B1. Bloomberg-style 정보 밀도 토글 (Compact / Standard / Spacious)**
- 1행 = 1종목(트레이더용 12px) vs 카드형(리테일용 16px) 즉시 전환
- 모바일에서는 자동으로 Compact, 데스크탑은 Standard 디폴트

**B2. Sparkline + Score Trajectory in-row (Renaissance: 시각적 패턴)**
- 종목 행 안에 **30일 가격 sparkline + 점수 변화선 2단 오버레이**
- 절대 숫자보다 모양(추세)이 먼저 읽히도록

**B3. Severity-based Color Semantics (Citadel risk dashboard)**
- 색을 "예쁨"이 아닌 **위험·기회 그라데이션**으로 일관 적용
  - Red → 손절 임박 / 모멘텀 꺾임
  - Amber → 관찰
  - Green → 진입 신호
  - Blue → 이미 보유(사용자 watchlist 연동)
- 색맹 대응: 색 + 아이콘(▲▼●) 동시

**B4. Progressive Disclosure for 한줄평**
- 1줄 요약 → 호버 시 3줄 코멘트 → 클릭 시 풀 카드(점수 분해·해자·재무·뉴스)
- 현재 모든 정보가 같은 깊이에 있어 인지부하 큼

**B5. Empty State → Onboarding Tour (Goldman Sachs Marquee 발상)**
- 첫 방문자에게 "워치리스트 3개 추천 → 알람 1개 설정 → 백테스트 1회 실행" 3-스텝 투어
- 빈 화면을 가치 전달의 기회로 전환

### C. Engineer — 기술 혁신·API·플랫폼

**C1. SSE/WebSocket 라이브 스트림 (Citadel low-latency)**
- 현재 폴링 → **Server-Sent Events**로 점수 변경분만 푸시
- 페이로드 1KB 미만, 사용자는 "지금 막 NVDA 점수가 올라갔다"를 즉시 체감

**C2. Client-side 뷰 가상화 (Two Sigma 대규모 데이터)**
- 500종목 동시 렌더 시 끊김 → `react-window`/`virtual-scroll` 패턴으로 보이는 50개만 DOM
- 현재 app.js 4,935줄에서 가상화 미적용 추정 → 즉시 효과 큼

**C3. Backtest-as-a-Service 패널 (Renaissance)**
- "이 필터 조합으로 지난 3년 백테스트" 버튼 → 비동기 잡 큐, 결과는 알람으로
- 사용자가 자기 가설을 검증하는 순환 고리

**C4. Saved Views + URL Encodable State**
- 필터·정렬·테마 조합을 URL에 인코딩 → 공유·북마크 가능
- 카카오톡 단톡방 공유 시 정확히 같은 화면 재현

**C5. Explainability API (BlackRock Aladdin 발상)**
- `/api/explain/<ticker>` — 점수가 왜 그런지 SHAP-style 기여도 분해 반환
- UI에서 "왜 이 점수인가?" 클릭 시 막대 그래프로 표시 (모멘텀 +12, 밸류 -5, 해자 +8...)
- 신뢰의 핵심 = 설명 가능성

## 3. 각 아이디어의 핵심 가정(테스트해야 할 것)

**A2 (델타 뷰)** — 가정: 사용자는 "오늘의 변화"를 가장 원한다. 반증 시나리오: 한국 리테일은 절대 점수 순위에 더 익숙해 델타가 혼란을 줄 수 있음 → A/B 토글 필수.

**C5 (Explainability)** — 가정: 점수 분해를 봐도 이해할 수 있는 사용자가 충분히 많다. 반증 시: "어차피 설명 봐도 모르겠다" → 그래도 신뢰는 올라간다는 2차 효과 측정.

**A3 (Confidence)** — 가정: ●/●●/●●● 3단계가 사용자 의사결정에 영향. 반증 시: 모든 등급을 똑같이 취급 → 등급별 클릭률·후속행동 차이 측정.

**B3+B2 (Color + Sparkline)** — 가정: 시각 단서가 숫자보다 빨리 읽힌다. 반증 시: 색맹/색 피로 → 아이콘 폴백 미리 설계.

**C1+A1 (SSE + Regime)** — 가정: 실시간 푸시가 가치 있고 인프라 비용 감당 가능. 반증 시: 동시접속이 적어 폴링과 차이 미미 → 50동접 임계 후 도입으로 보류.

## 5. 무엇을 하지 말 것 (Anti-list)

- 풀 리뉴얼/리브랜딩 — 가치 검증 없는 디자인 작업은 비용만 들고 retention에 무영향.
- 차트 라이브러리 교체 — 현재 sparkline 정도면 충분, TradingView 임베드는 사용자 1,000명 이상에서 ROI.
- 모바일 네이티브 앱 — PWA + 반응형 우선. 네이티브는 검증된 retention 곡선 이후.
- "AI 챗봇" 추가 — 한줄평이 이미 그 역할. 챗봇은 신뢰도 낮은 답을 양산할 위험.
