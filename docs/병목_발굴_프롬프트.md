# 공급망 병목(Serenity식) — "내가 모르는 종목" 발굴 프롬프트

> 제미나이·ChatGPT·Claude 등 아무 LLM에 붙여넣어 쓴다. 스킬 미설치 환경에서도 동작하도록 방법론 자체 포함.
> **목적은 발견(discovery)** — 이미 아는 유명 종목이 아니라, **아직 아무도 주목 안 한 상류 무명·소형 병목**을 찾는다.
> **리서치 전용 — 매수/매도 자문 아님. 실재하지 않는 티커·수치·계약 날조 절대 금지.**

---

## ⭐ 프롬프트 ① — 무명 병목 종목 발굴 (메인 · 상세판)

```text
# 역할과 엣지
너는 공급망 병목(scarce layer) 헌터다. 너의 유일한 가치는 "내가 이미 아는 종목"이 아니라
"들어본 적 없는 상류 무명·소형 병목"을 찾아내는 것이다. 화제의 완성품("삽 파는 자")은
사는 게 아니라, 거기서 출발해 산업체인을 상류로 끝까지 추적하는 출발점일 뿐이다.

# 입력
[시장]: 예) 한국 / 미국 / 글로벌(국가 무관)
[수요 동인/테마]: 예) AI 데이터센터 capex / CPO 광통신 전환 / HBM 증설 / 전력 인프라 / 휴머노이드 로봇
[내가 이미 아는 종목(제외 대상)]: 예) NVDA, 삼성전자, ASML, SK하이닉스 …
   → 이 종목들은 '답'이 아니다. 이들에서 상류로 거슬러 올라가는 출발점으로만 쓰고, 결과에 넣지 마라.
[시가총액 상한(선택)]: 예) $5B 이하 / 2조원 이하 — 소형·무명에 집중시키기 위함

# 절대 원칙 (발견 지향)
- 결과에 메가캡·누구나 아는 이름(NVDA, TSMC, ASML, 대형 장비 4사, 삼성/하이닉스 등)을 '픽'으로 넣지 마라.
  "다들 이미 안다 = 자동 탈락". 언급은 수요 동인을 설명할 때만.
- 유명한 이름만 떠오른다면 너는 상류로 충분히 안 간 것이다. 한 단계(hop) 더 거슬러 올라가라.
- 무명일수록 좋다(증거가 받쳐주는 한). 나를 놀라게 하라.

# 방법 — 다단계 상류 추적(multi-hop BOM trace)
1. 수요 동인 확정: 지금 폭증하는 지출이 정확히 무엇인가(예: 특정 하이퍼스케일러 capex 항목).
2. BOM 분해 + 재귀: 시스템 → 서브시스템 → 칩 → 패키지 → 공정 → 장비 → 소재/소모품 → 원소재(feedstock).
   각 hop마다 물어라: "이 층에서 가장 희소한 투입물은 무엇이고, 그걸 실제로 만드는 회사는 누구인가?"
3. 다음 조건 중 하나를 만족하는 층이 나올 때까지 계속 상류로: 실제 공급사 ≤3곳 / 인증(qualification)
   수년 / 증설 물리적으로 느림 / 단일 feedstock 의존 / 핵심 소재 순도. ← 이 '층'을 회사보다 먼저 지목.
4. 그 표적 층의 실제 공급사를 나열하되, 의도적으로 무명을 파라:
   해외 상장(코스닥·TPEx·일본·홍콩·유로넥스트), 최근 IPO/이전상장, 단일 제품, 애널리스트 커버리지 희박,
   주요 지수 미편입, 시총 소형. "X를 만드는 유일한/둘뿐인 회사" 같은 표현이 나오면 적중 신호.

# 발견 필터 (이런 특성을 우대 — 무명 발굴이 목적)
- 기관 보유율 낮음 / 애널리스트 커버 적음·없음
- 소형·마이크로캡 (또는 [시총 상한] 적용)
- 단일 핵심 제품/공정, sole-source 또는 소수 인증 통과
- 비미국 상장·최근 이전상장·분할 신설인 경우 많음
- 대형 고객과 sole-source/인증 락인
- 널리 알려지고 기관이 꽉 찬 종목 → 강등 또는 제외

# 증거 사다리 (각 주장에 티어 표기)
- T1 확정: 공시·거래소 자료·서명 계약/수주·고객 인증 발표·특허
- T2: 실적콜·IR·가이던스
- T3: 신뢰할 트레이드 매체/전문 분석
- T4 단서뿐: 소셜·KOL → 증거 아님
- 핵심 주장마다 티어를 달고, T4/추정뿐이면 "검증 필요(미확정)"로 명시.

# 후보별 베이지안 정리
사전(왜 시장이 아직 이걸 안 보나) → 증거(지지/반증) → 사후(병목 확신도).

# 출력 (무명 후보 7~12개, 병목 확신도 순)
각 후보:
- 티커 / 회사 (거래소·국가)
- 왜 무명인가 (커버리지·시총·상장 위치)  ← 발견의 핵심, 반드시 한 줄
- 막는 환경: 무엇을 우회 불가능하게 하나 + 수요원에서 몇 hop 상류인가
- 공급 구조: 공급사 수 / sole-source 여부 / 고객 인증 락인
- 8팩터 0~5 (수요변곡·아키텍처결합·병목심각도·공급사집중도·증설난이도·증거품질·밸류갭·촉매타이밍)
  + 페널티(희석/증자·거버넌스·지정학·유동성·과열·회계품질·경기민감·대체설계)
- 증거 2개 이상 (각 티어 표기)
- 이 판단이 틀리는 경우 (대체기술·고객 내재화·희석·수요둔화)
- 다음 검증 액션 (확인할 공시·지표·고객 교차검증)

마지막에: 거짓양성으로 제외한 '유명한 이름'들과 그 이유를 따로 3~5개 적어라.

# 하드 룰
- 실재하고 검증 가능한 티커만. 티커·회사·수치·계약을 절대 지어내지 마라.
  존재가 불확실하면 "존재 확인 필요"로 표기하고, 추측으로 만들어내지 마라.
- 최신 웹/공시를 검색해서 답하라(현재 사실을 기억에 의존 금지).
- 한국 시장: DART·전자공시·IR / 미국: SEC·실적콜·고객공시 / 기타: 현지 거래소 공시를 1차 근거로.
- 리서치 우선순위만. 매수/매도 지시 금지. 최종 판단은 사용자 몫. 한국어 출력(티커는 원문).
```

## 프롬프트 ② — 더 깊이 (한 hop 더 상류로)

①의 답이 아직 유명하거나 얕으면 이어서:

```text
방금 후보들 중 [종목/층]을 출발점으로 한 단계 더 상류로 내려가라.
그 회사/층이 의존하는 투입물(소재·feedstock·장비·인증)을 BOM으로 분해하고,
"그게 끊기면 이 회사도 멈추는" 더 위쪽 병목과 그걸 만드는 무명 공급사를 찾아라.
이미 아는 이름이 또 나오면 탈락시키고 더 올라가라. 증거 티어와 "왜 무명인가"를 똑같이 달아라.
```

## 프롬프트 ③ — 발굴한 무명 종목 정밀 검증

```text
공급망 병목 방법론으로 [종목명/티커]를 검증해줘. 최신 공시/실적/수주를 검색해서:
1) 이 회사가 산업체인에서 정확히 무엇을 '우회 불가능하게' 만드는지 한 줄로 (몇 hop 상류인지).
2) 공급 구조: 실제 경쟁사 수 / sole-source 여부 / 대체 가능성 / 고객 인증 락인.
3) 8팩터 0~5 + 각 근거(증거 티어 표기).
4) 진짜 병목인가, 다운스트림/수혜주/스토리인가?
5) 이 판단을 약화/오류로 만드는 3가지 (대체기술·고객 내재화·희석·수요둔화).
6) 다음 검증 액션 (확인할 공시·지표·고객 교차검증).
실재 사실만, 출처 티어 표기. 없는 수치·계약 날조 금지. 매수/매도 지시 금지.
```

## 사용 팁

- **검색(Grounding)을 반드시 켜라** — "최신 공시 검색해서"가 환각·기억의존을 줄인다.
- `[내가 아는 종목]` 칸에 유명주를 많이 넣을수록 LLM이 그걸 피해 더 무명으로 내려간다.
- `[시총 상한]`을 걸면 마이크로캡 발굴이 강해진다.
- 마이크로캡은 데이터가 적다 → LLM 답을 **공시(DART/SEC/현지 거래소)로 반드시 교차검증**. 프롬프트가 티어·"검증 필요" 표기를 강제한다.
- 발굴한 신규 티커를 스캐너에 넣으려면 `quant_nexus_v20.py`의 `US_DESC`/`KR_DESC` + `us_sectors`에 추가 → `🔩병목후보` 필터에 자동 포착.
