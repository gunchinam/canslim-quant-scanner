# 공급망 병목(Serenity식) — "내가 모르는 종목" 발굴 프롬프트 (심층판)

> **역할 분담 결론**: 발굴(유니버스 밖 무명 종목)은 스캐너가 구조적으로 못 한다 → **LLM이 발굴**.
> 발굴한 티커를 스캐너에 넣으면 **타이밍·리스크·병목 배지·IC로 모니터링**(스캐너의 일).
> 이 문서는 **발굴용 LLM 프롬프트**다. 제미나이·ChatGPT·Claude 어디든 검색 켜고 붙여넣어 쓴다.
>
> **리서치 전용 — 매수/매도 자문 아님. 실재하지 않는 티커·수치·계약 날조 절대 금지.**

---

## ⭐ 프롬프트 ① — 무명 병목 종목 발굴 (메인, 풀버전)

아래 블록을 통째로 붙여넣고 `[ ]` 칸만 채운다.

```text
# 역할
너는 공급망 병목(scarce layer) 헌터다. 너의 유일한 가치는 "이미 유명한 종목"이 아니라,
산업체인을 끝까지 거슬러 올라가야 나오는 "들어본 적 없는 상류 무명·소형 병목"을 찾아내는 것이다.
시스템이 확장될 때 "이게 막히면 다운스트림 전체가 멈추는데, 우회할 방법이 없는" 가장 좁은 길목 = 병목.
화제의 완성품(엔비디아 같은 '삽 파는 자')은 사는 게 아니라, 거기서 출발해 위로 추적하는 출발점일 뿐이다.

# 입력
[시장]: 예) 미국 / 한국 / 글로벌(국가 무관, 비미국 상장 적극 포함)
[수요 동인]: 예) AI 데이터센터 capex / CPO 광통신 전환 / HBM 증설 / 전력·냉각 인프라 / 휴머노이드 로봇 / 방산
[내가 이미 아는 종목(결과에서 제외)]: 예) NVDA, AMD, TSMC, ASML, 삼성전자, SK하이닉스, 마이크론 ...
[시가총액 상한(권장)]: 예) $5B 이하 / 3조원 이하  ← 무명·소형에 강제 집중
[기간 가정]: 예) 향후 12~24개월 수요 가시성

# 절대 원칙 (발견 지향 — 어기면 실패)
1) 결과 픽에 메가캡·누구나 아는 이름을 넣지 마라. "다들 이미 안다 = 자동 탈락."
   그런 이름은 오직 '수요 동인'을 설명할 때만 등장한다.
2) 유명한 이름만 떠오르면 너는 상류로 충분히 안 간 것이다. 한 단계(hop) 더 올라가라.
3) 무명일수록 좋다(증거가 받쳐주는 한). 나를 놀라게 하는 게 목표다.
4) 추측으로 종목을 지어내지 마라. 실재 확인 안 되면 "존재 확인 필요"로 표기.

# 방법 — 다단계 상류 추적 (multi-hop BOM trace)
STEP 1. 수요 동인 고정: 지금 폭증하는 '지출 항목'을 구체적으로 적어라(예: 하이퍼스케일러의 800G→1.6T 광모듈 전환).
STEP 2. BOM 분해 + 재귀 하강: 아래 사슬을 한 칸씩 내려가며, 각 칸마다 질문하라 —
        "이 층에서 가장 희소한 투입물은 무엇이고, 그걸 '실제로 만드는' 회사는 누구인가?"
        [완성 시스템] → [서브시스템/모듈] → [칩/디바이스] → [패키지/후공정] → [공정] →
        [장비/검사] → [소재/소모품] → [원소재·feedstock]
STEP 3. 정지 조건(아래 중 하나 만족 시 그 '층'이 표적):
        · 실제 공급사 ≤ 3곳   · 인증(qualification) 수년 소요   · 증설이 물리적으로 느림(클린룸·결정성장·정제)
        · 단일 feedstock/원소재 의존   · 핵심 소재 순도/수율 know-how   · 고객 인증 락인(designed-in)
        먼저 '층'을 지목하고 그 다음에 회사를 댄다(회사보다 층이 먼저).
STEP 4. 표적 층의 실제 공급사 열거 — 의도적으로 무명을 파라:
        해외 상장(코스닥·TPEx·일본·홍콩·유로넥스트), 최근 IPO/이전상장/분할 신설, 단일 제품 집중,
        애널리스트 커버리지 희박, 주요 지수 미편입, 시총 소형. "X를 만드는 유일한/둘뿐인 회사" = 적중 신호.
STEP 5. 후보가 또 유명하면 STEP 2로 돌아가 한 hop 더 올라가라(예: 레이저가 유명하면 → 레이저용 화합물 기판 → 그 기판의 원소재 정제).

# 발견 휴리스틱 (무명이 숨어있는 곳)
- 기관 보유율·애널 커버 낮음 / 시총 소형(또는 [시총 상한] 적용)
- 단일 핵심 공정·소재 / sole-source / 고객 집중(리스크가 아니라 '락인'의 증거일 수 있음)
- 비미국 상장이 잦음(미국 ETF·패시브에 안 잡혀 가격 공백) / 최근 상장이라 히스토리 짧음
- 레거시 사업 매각하고 병목 사업으로 전환 중인 회사
- 널리 알려지고 기관이 꽉 찬 종목 → 강등 또는 제외

# 증거 사다리 (주장마다 티어 표기 — 환각 방지의 핵심)
- T1 확정: 공시/거래소 자료/서명 계약·수주/고객 인증 발표/특허/규제 문서
- T2: 실적콜·IR·가이던스·투자설명회
- T3: 신뢰할 트레이드 매체·전문 분석
- T4 단서뿐: 소셜·KOL·루머 → 증거가 아니라 '리드'일 뿐
- 핵심 주장마다 [T1]~[T4] 표기. T4/추정만 있으면 반드시 "검증 필요(미확정)".

# 후보별 베이지안 정리
사전(왜 시장이 아직 이걸 안 보나) → 증거(지지/반증, 티어 표기) → 사후(병목 확신도 1~5).

# 8팩터 스코어카드 (각 0~5, 정의 그대로 적용)
1) 수요변곡(demand_inflection): 수요가 선형이 아니라 변곡(가속)했는가. 구조적 신규 수요인가.
2) 아키텍처결합(architecture_coupling): 신규 시스템 설계에 이 부품이 '필수로 박혀'(designed-in) 떼기 어려운가.
3) 병목심각도(chokepoint_severity): 이게 막히면 다운스트림이 실제로 멈추는가. 우회 경로가 없는가.
4) 공급사집중도(supplier_concentration): 진짜 공급사가 몇 곳인가(독점/과점/경쟁). 적을수록 높음.
5) 증설난이도(expansion_difficulty): 캐파를 빨리 못 늘리는가(인증·결정성장·정제·클린룸·장비 리드타임).
6) 증거품질(evidence_quality): 위 주장들이 T1~T2 1차 증거로 뒷받침되는가.
7) 밸류갭(valuation_disconnect): 병목 지위 대비 시장이 아직 싸게(또는 못 알아보고) 매기는가.
8) 촉매타이밍(catalyst_timing): 향후 12개월 내 재평가 트리거(수주·인증·실적·정책)가 있는가.
페널티(감점, 각 0~5): 희석/증자 · 거버넌스 · 지정학 · 유동성 · 과열(이미 폭등) · 회계품질 · 경기민감 · 대체설계 위험.

# 출력 형식
먼저 한 단락: "수요 동인 → 어느 층이 병목인지(층 랭킹)" 를 평이한 말로 요약.
그 다음 무명 후보 7~12개를 병목 확신도 순으로, 각 항목:
- 티커 / 회사 (거래소·국가)
- ★ 왜 무명인가 (커버리지·시총·상장 위치) — 반드시 한 줄, 발견의 핵심
- 막는 환경: 무엇을 우회 불가능하게 하나 + 수요원에서 몇 hop 상류인가
- 공급 구조: 실제 공급사 수 / sole-source 여부 / 고객 인증 락인
- 8팩터 점수(0~5 나열) + 합산 + 주요 페널티
- 증거 2개 이상 (각 [T1]~[T4] 표기)
- 이 판단이 틀리는 경우 3가지 (대체기술·고객 내재화·희석·수요둔화 중)
- 다음 검증 액션 (확인할 공시·지표·고객 교차검증)
마지막: '거짓양성으로 제외한 유명한 이름' 3~5개 + 제외 이유.

# 답하기 전 자기점검 (반드시 통과)
□ 픽에 메가캡·누구나 아는 이름이 0개인가? (있으면 제거하고 더 상류로)
□ 각 후보에 "왜 무명인가"가 적혔는가?
□ 각 핵심 주장에 증거 티어가 붙었는가?
□ 모든 티커가 실재하는가? 불확실하면 "존재 확인 필요"로 표기했는가?
□ 회사보다 '층'을 먼저 랭킹했는가?

# 하드 룰
- 실재·검증 가능한 티커만. 티커·회사·수치·계약 날조 절대 금지.
- 최신 웹/공시 검색해서 답하라(현재 사실을 기억에 의존 금지).
- 한국: DART·전자공시·IR / 미국: SEC·실적콜·고객공시 / 기타: 현지 거래소 공시.
- 리서치 우선순위만. 매수/매도 지시 금지. 최종 판단은 사용자 몫. 한국어 출력(티커는 원문).
```

## 프롬프트 ② — 한 hop 더 상류로 (재귀 심화)

①의 후보가 아직 유명하거나 얕으면 이어서:

```text
방금 후보 중 [종목/층]을 출발점으로, 그게 의존하는 투입물(소재·feedstock·장비·인증)을 BOM으로 분해해라.
"그게 끊기면 이 회사도 멈추는" 더 위쪽 병목과, 그걸 만드는 더 무명인 공급사를 찾아라.
이미 아는 이름이 또 나오면 탈락시키고 한 칸 더 올라가라.
각 후보에 "왜 무명인가" + 증거 티어 + 8팩터를 똑같이 달아라.
```

## 프롬프트 ③ — 발굴 종목 정밀 검증 (반증 중심)

발굴한 무명 종목을 그대로 믿지 말고 깨보기:

```text
공급망 병목 방법론으로 [종목명/티커]를 '반증 중심'으로 검증해줘. 최신 공시/실적/수주 검색해서:
1) 막는 환경 한 줄(몇 hop 상류인지).
2) 공급 구조: 실제 경쟁사 수 / sole-source 여부 / 대체 가능성 / 고객 인증 락인.
3) 8팩터 0~5 + 각 근거(증거 티어).
4) 진짜 물리적 병목인가, 아니면 IP·서비스·다운스트림·스토리인가? (예: IP 라이선스는 물리적 증설 병목이 아님)
5) 이 thesis를 죽이는 가장 강한 반대 시나리오 3가지 + 각각을 확인할 데이터.
6) 지금 이미 몇 배 올랐나? (오버크라우드/과열이면 '발굴'이 아니라 '늦음').
실재 사실만, 출처 티어 표기. 없는 수치·계약 날조 금지. 매수/매도 지시 금지.
```

## 프롬프트 ④ — 비교 랭킹 (발굴 후 줄세우기)

```text
공급망 병목 관점에서 다음 무명 후보들을 비교 랭킹: [A, B, C ...]
기준: ①막는 환경(체인 위치, 상류일수록 우위) ②공급사 집중도 ③증설 난이도 ④증거 강도(티어)
⑤밸류갭 ⑥촉매 타이밍 ⑦핵심 리스크 ⑧이미 오른 정도(과열).
표로 줄세우고, 1순위에 "이 판단이 틀리는 경우"를 반드시 적어줘. 매수/매도 지시 금지.
```

## 운영 루프 (발굴 → 스캐너 모니터링)

1. **발굴**: 프롬프트 ①로 무명 후보 리스트 확보 → ③으로 반증 검증 → 살아남은 티커 추림.
2. **편입**: 살아남은 티커를 `quant_nexus_v20.py`의 `US_DESC`/`KR_DESC` + `us_sectors`에 추가
   (설명에 병목 키워드 포함: 기판/substrate·에피·레이저·CMP·식각·후공정·HBM·CPO·전력반도체 등).
3. **모니터링**: 스캐너가 자동으로 부착 —
   - `🔩병목후보`/`🎯병목+진입` 퀵필터, 종목명 옆 병목 배지(🟢/🟡/⚪)
   - **진입 게이트**: 이미 폭등(3M+80%)·과매수면 🟡조정대기로 자동 분리(고점 추격 방지)
   - **forward IC**: 등급 스냅샷 누적 → 실제 예측력 검증
4. **재검증**: 분기마다 프롬프트 ②로 한 hop 더 파서 새 무명 노드 갱신.

> 왜 이 구조인가: **발굴은 유니버스 밖 추론**이라 LLM의 일이고, **타이밍·리스크·지속 모니터링은 정량**이라
> 스캐너의 일이다. 키워드 스캐너 하나로 발굴까지 하려던 게 무리였다(유니버스 밖을 못 보고, 키워드 오탐).

## 사용 팁
- **검색(Grounding) 필수** — "최신 공시 검색해서"를 꼭 넣어라(환각·기억의존 감소).
- `[내가 아는 종목]`에 유명주를 많이 넣을수록 LLM이 그걸 피해 더 무명으로 내려간다.
- `[시총 상한]`을 걸면 마이크로캡 발굴이 강해진다.
- 마이크로캡은 데이터가 적다 → LLM 답을 **공시(DART/SEC/현지거래소)로 반드시 교차검증**.
- 첫 답이 유명주 위주면 "전부 너무 유명하다, 두 hop 더 올라가라"고 한 번 더 밀어붙여라.
