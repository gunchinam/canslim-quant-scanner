> 작성일: 2026-06-06

## 1. 현황 요약 (앱이 지금 제공하는 것)

- **이중 스캔 엔진**: US(yfinance) + KR(네이버·DART·yfinance) 종목을 CAN SLIM·4축·스캘핑 전략으로 점수화. `/api/scan`은 pickle 캐시 + stale-while-revalidate + 사전 gzip 압축으로 최적화됨.
- **상세 분석 패널**: 4축 카드, 기술/재무 탭, RS Rating 도넛, GreedZone 과열 게이지, 드로다운/MDD 리스크 카드, 섹터 분류, 해자(moat) 가산점, 증권사 목표가(네이버), 한줄평(주갤 톤 2,600+ 풀).
- **시장 맥락**: 상단 매크로 신호등 띠(VIX·환율·금리·KOSPI·S&P), ETF 현황 탭(미·한 인기 ETF), 매크로 이벤트.
- **종목별 API 다수**: sentiment, investor_flow, peers, segments, events, insider, ownership, consensus, regime, four_axis, dart-news, us-insight, score-history, bucket-stats.
- **실행 보조 일부**: watchlist(서버 SQLite 영속화), 12개 퀵필터, 종목 비교, CSV 내보내기, 공유 카드(PNG), 점수 히스토리(어제 대비 점수·순위 델타·신규 진입).
- **부가**: 실시간 익명 채팅(WebSocket + 관리자), 로딩 trivia.
- **미연결 자산(코드는 있으나 라우트/UI 미노출)**: `position_sizer.py`(포지션 사이징 완성), `notifier.py`(토스트 알림 백엔드 완성), `event_calendar.py`(이벤트 캘린더), `valuation_engine.py`, `backtest/`(엔트리 타이밍 백테스트 스크립트군).

## 2. 기능 제안 목록

### P0 — 지금 당장 (효과 높음 × 난이도 낮음, 기존 자산 재활용)

#### F1. 포지션 사이징 위젯 (상세 패널 내장)
- **한 줄**: 계좌·리스크%·진입가·손절가 입력 → 권장 수량·금액·R:R·VIX 자동 축소 표시.
- **사용자 가치**: "몇 주 사야 안전한가"를 화면 안에서 즉답. ROADMAP의 핵심 미충족 항목(C1).
- **난이도**: 하 (반나절). `position_sizer.py`가 이미 완성·테스트됨(`calc_position`, `calc_rr`, `suggest_targets`).
- **재활용**: `position_sizer.py` 그대로 + `/api/four_axis`나 상세 데이터의 진입가·ATR·손절가 + `/api/macro`의 VIX. 신규 라우트 `/api/position` 1개 + 진입플랜 탭 입력 UI.
- **데이터 소스**: 전부 내부 데이터. 외부 의존 0.
- **계획 중복**: ROADMAP v21 A3/C1에 명시됨(미구현).

#### F2. 가격·점수 알림 (관심종목 조건 도달 토스트/텔레그램)
- **한 줄**: watchlist 종목이 목표가·손절가·점수 임계 도달 시 토스트 + 텔레그램 푸시.
- **사용자 가치**: 화면을 떠나도 가치가 나옴 — ROADMAP 비전("발굴→실행→추적")의 핵심.
- **난이도**: 중 (1~2일). `notifier.py` 토스트 백엔드 완성됨. 텔레그램은 README에 키 설정 자리 존재.
- **재활용**: `notifier.py` + 기존 watchlist SQLite + `_refresh_scan_background` 백그라운드 워커 패턴. 조건은 watchlist 행에 note 대신 rule 컬럼 추가.
- **데이터 소스**: 내부 스캔 결과. 텔레그램 봇 토큰(무료).
- **계획 중복**: ROADMAP A2에 명시됨(미구현).

#### F3. 이벤트 캘린더 D-Day 칩 (실적·공시 임박 경고)
- **한 줄**: 종목명 옆 "실적 D-2" 칩 — 실적 발표/배당락 임박 종목 시각 경고.
- **사용자 가치**: 실적 직전 진입의 갭 리스크를 사전 차단(안전장치).
- **난이도**: 중 (1일). `event_calendar.py` + `/api/events/<ticker>` 라우트가 이미 존재 — 목록 행에 노출만 안 됨.
- **재활용**: `/api/events` 응답을 스캔 결과에 머지 → 행 렌더(`renderStockRow`)에 칩 추가. 테마주 ⚠ 칩과 동일 패턴.
- **데이터 소스**: Finnhub 어닝 캘린더(무료 60/min, 키 이미 지원), DART(KR).
- **계획 중복**: ROADMAP C2에 명시됨(부분 — API는 있으나 목록 노출 미구현).

#### F4. 어제 대비 변동 정렬·필터 (이미 있는 델타 데이터 활용)
- **한 줄**: "점수 급등 TOP" / "순위 신규 진입" 정렬·퀵필터 추가.
- **사용자 가치**: 매일 무엇이 바뀌었나를 한눈에 — 재방문 동기 부여.
- **난이도**: 하 (2~3시간). `history.annotate_deltas`가 이미 `ScoreDelta/RankDelta/IsNew`를 모든 행에 주입 중. 현재 UI에서 거의 안 쓰임.
- **재활용**: 기존 퀵필터 칩 패턴(`data-filter`) + 정렬 로직에 `ScoreDelta` 키 추가. 백엔드 변경 0.
- **데이터 소스**: 내부 스냅샷(이미 매 스캔 저장 중).
- **계획 중복**: 없음. 사실상 "공짜" 기능.

### P1 — 곧 (효과 높음 × 난이도 중간)

#### F5. 거래 일지 / 트레이드 로그
- **한 줄**: 관심종목에서 "매수 기록" → 진입가·사유·당시 4축 점수 자동 저장, 추후 손익 추적.
- **사용자 가치**: 발굴→실행→**검증** 루프 완성. 자기 매매 복기.
- **난이도**: 중 (2일). watchlist SQLite 패턴(`WatchlistDB`)을 그대로 복제해 `trade_log` 테이블.
- **재활용**: `swing_scan/journal/` 디렉터리가 이미 존재(저널 개념 선례). watchlist API CRUD 패턴 복붙.
- **데이터 소스**: 내부. 종가 추적은 yfinance.
- **계획 중복**: ROADMAP A4에 명시됨(미구현). 단 README "자동매매 금지" 원칙 — 기록 전용이라 안전.

#### F6. Piotroski F-Score (재무 건전성 9점)
- **한 줄**: 9개 이진 재무 룰 합산 → 회사 펀더멘털 등급(0~9).
- **사용자 가치**: "좋은 회사인가"를 객관 점수로 보강. CAN SLIM 탭 깊이 강화.
- **난이도**: 중 (1일). 재무 데이터는 `us_company_info.py`/`naver_quarter.py`/`valuation_engine.py`에 이미 수집됨.
- **재활용**: `valuation_engine.py`·`four_axis_analyzer.py`의 재무 파싱 + 재무 탭 UI(`/api/ticker`).
- **데이터 소스**: yfinance financials(US), 네이버/DART(KR) — 둘 다 이미 연동. 실현성 높음.
- **계획 중복**: ROADMAP B2에 명시됨(미구현).

#### F7. FRED 거시 게이트 (Risk-Off 시 진입 보류 배너)
- **한 줄**: VIX·HY 스프레드·10Y-2Y 종합 → Risk-Off면 헤더에 "신규 진입 신중" 배너.
- **사용자 가치**: 시장 전체가 위험할 때 개별 종목 초록불을 한 번 걸러줌(안전장치).
- **난이도**: 중 (반나절~1일). `macro.py`가 이미 VIX 수집 + TTL 캐시 패턴 완비. `macro_gate.py` 스텁 존재.
- **재활용**: `macro.py` 구조 복제 + 기존 매크로 신호등 띠 UI에 게이트 레벨 1개 추가.
- **데이터 소스**: FRED API(무료, 키 발급), 일부는 yfinance(^VIX, ^TNX)로 키 없이 가능.
- **계획 중복**: ROADMAP B3에 명시됨(미구현).

#### F8. 멀티 타임프레임 추세 일치도 (일·주봉 컨펌)
- **한 줄**: 일봉+주봉 추세 방향 일치 시 ★★★ 배지 — 추세 신뢰도 표시.
- **사용자 가치**: 단일 타임프레임 함정(일봉만 좋고 주봉 깨짐) 회피.
- **난이도**: 중 (1~2일). 스캔이 이미 1년치 일봉을 받음(`yf.download period=1y`) → 주봉 리샘플만 추가.
- **재활용**: GreedZone batch download 패턴(app.py:686) 그대로. MA/추세 판정은 엔진 내 기존 함수.
- **데이터 소스**: 내부(추가 호출 없이 리샘플).
- **계획 중복**: ROADMAP B4에 명시됨(미구현).

### P2 — 나중 (효과 중간 또는 난이도 높음)

#### F9. 섹터 로테이션 히트맵 (ETF 탭 확장)
- **한 줄**: 11개 US 섹터 ETF + KR 업종 ETF의 1주/1개월/3개월 상대강도 히트맵.
- **사용자 가치**: "지금 어느 섹터에 돈이 도나"를 한 화면에 — 톱다운 종목 선정.
- **난이도**: 중 (1일). `etf.py`가 이미 섹터 ETF(XLK/XLF/XLE/XLV/SMH) 수집 + 52주 위치 계산.
- **재활용**: `etf.py`에 멀티 기간 수익률 컬럼 추가 + ETF 탭 UI에 색상 그리드. 신규 데이터 소스 0.
- **데이터 소스**: 내부(이미 1년치 받음).
- **계획 중복**: 없음. ETF 탭의 자연스러운 확장.

#### F10. 백테스트 미니 카드 (진입 신호 과거 적중률)
- **한 줄**: 상세 패널에 "이 진입 신호의 과거 N회 중 승률·평균 R" 표시 (과거성과 한정 표기).
- **사용자 가치**: 신호에 대한 신뢰 근거. 단, 검수자 원칙상 "기대수익률" 표기 금지.
- **난이도**: 상 (2~3일). `backtest/entry_timing_backtest.py` 등 스크립트 자산 존재하나 웹 통합 필요.
- **재활용**: `backtest/` 모듈 + 점수 히스토리 스냅샷(`snapshots/`)을 라벨 데이터로.
- **데이터 소스**: 내부 과거 데이터. 표본 적으면 신뢰 낮음 — 솔직히 표본 누적 후 가치.
- **계획 중복**: ROADMAP Sprint4 "백테스트 PoC"에 언급. 단 README 금지선("기대수익률 표기") 준수 필요.

#### F11. 종목 비교 레이더 차트 강화
- **한 줄**: 현재 비교 화면에 4축 레이더(오버레이) 추가 — 2~4종목 강약 즉시 대조.
- **사용자 가치**: 표 숫자 대조보다 직관적. 비교 기능 사용성 향상.
- **난이도**: 중 (1일). `/compare` 라우트·비교 UI 이미 존재. SVG 레이더는 RS 도넛(UX개선안 B2)과 동일 무라이브러리 패턴.
- **재활용**: `_renderQuadrant`/RS 도넛 SVG 패턴 + `/api/four_axis` 데이터.
- **데이터 소스**: 내부.
- **계획 중복**: 없음(UX개선안은 단일 종목 사분면만 다룸).

#### F12. PWA / 홈 화면 추가 + 오프라인 캐시
- **한 줄**: manifest + service worker로 모바일 홈 아이콘 설치·마지막 스캔 오프라인 표시.
- **사용자 가치**: 모바일 재방문 마찰 감소(앱처럼). 모바일 카드 UI가 이미 잘 다듬어져 있음.
- **난이도**: 중 (1일). 정적 파일 2개 + Flask 정적 라우트.
- **재활용**: 기존 gzip 스캔 캐시 응답을 SW가 캐싱. UI 변경 최소.
- **데이터 소스**: 불필요.
- **계획 중복**: 없음.

#### F13. KR 수급 주체별 매매 추세 (외국인·기관 N일)

## 3. 우선순위 매트릭스 (효과 × 난이도)

```
효과
높음 │ F1 F2 F3 F4        │ F5 F6 F7 F8        │ F10
     │ (지금 당장)         │ (곧 — 1~2일)        │ (백테스트, 상)
─────┼────────────────────┼────────────────────┼──────────────
낮음 │ F12 (PWA)          │ F9 F11 F13 F15     │ F14
     │                    │ (나중)              │ (탐색적)
     └────────────────────┴────────────────────┴──────────────
       난이도 낮음           난이도 중             난이도 높음
```

- **지금 당장 (좌상단)**: F1, F2, F3, F4 — 이미 작성된 모듈(position_sizer, notifier, event_calendar)을 라우트/UI에 연결만 하면 됨. ROI 최고.

## 5. 추천 Top 3

### 🥇 F1. 포지션 사이징 위젯
이유: `position_sizer.py`가 이미 완성·자체 테스트까지 통과한 상태. 라우트 1개 + 상세 패널 입력 UI만 붙이면 됨 (반나절). ROADMAP이 1순위(EXECUTION)로 못 박은 항목이고, "몇 주 살까"는 리테일이 매번 손으로 계산하는 마찰점. 투자 결정에 직접 닿는데 비용은 가장 낮음.

### 🥈 F4. 어제 대비 변동 정렬·필터
이유: `history.annotate_deltas`가 이미 모든 스캔 행에 `ScoreDelta/RankDelta/IsNew`를 주입하고 있는데 UI가 이를 거의 안 쓴다 — 사실상 "공짜"로 깔린 데이터. 퀵필터 칩 1~2개 + 정렬 키 추가(2~3시간, 백엔드 변경 0)로 "오늘 뭐가 바뀌었나"라는 재방문 동기를 만든다.

### 🥉 F3. 이벤트 캘린더 D-Day 칩
이유: `/api/events/<ticker>` 라우트와 `event_calendar.py`가 이미 있는데 목록에 노출이 안 됨. 실적 직전 진입의 갭 리스크는 초보가 가장 자주 당하는 사고이고, 테마주 ⚠ 칩과 동일한 렌더 패턴으로 1일 내 안전장치를 추가할 수 있다. 데이터 소스(Finnhub/DART)도 이미 연동됨.
